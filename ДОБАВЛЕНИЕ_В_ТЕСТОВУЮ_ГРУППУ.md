# Добавление бота в тестовую группу

## Шаг 1: Создайте тестовую группу (если еще нет)

1. Откройте Telegram
2. Создайте новую группу
3. Добавьте бота @tradetherapybot в группу
4. Сделайте бота администратором группы (для полной функциональности)

## Шаг 2: Получите ID тестовой группы

### Способ 1: Через @userinfobot
1. Добавьте бота **@userinfobot** в вашу тестовую группу
2. Отправьте в группу любое сообщение
3. Бот покажет ID группы (формат: `-1001234567890`)

### Способ 2: Через @RawDataBot
1. Добавьте бота **@RawDataBot** в группу
2. Отправьте `/start`
3. Найдите `"chat":{"id":-1001234567890}` — это ID группы

## Шаг 3: Добавьте ID группы в env_vars.txt на сервере

```bash
ssh root@81.200.146.32
cd /opt/tradetherapybot
nano env_vars.txt
```

Добавьте или измените строку:
```
BOT_TOKEN=8585081841:AAFbEB8LAKB5yT_YPpEg0WxbPcZFxz5ZiLs
ADMIN_ID=135366416
GROUP_CHAT_ID=-1001234567890
```

**Замените `-1001234567890` на реальный ID вашей тестовой группы!**

Сохраните: `Ctrl+O`, `Enter`, `Ctrl+X`

## Шаг 4: Перезапустите бота

```bash
systemctl restart tradetherapybot.service
systemctl status tradetherapybot.service
```

## Шаг 5: Проверьте работу

1. Отправьте `/start` боту
2. Попробуйте вступить в тестовую группу
3. Проверьте, что бот обрабатывает заявки (если включено одобрение заявок)

## Когда будет готова боевая группа

Просто замените `GROUP_CHAT_ID` в `env_vars.txt` на ID боевой группы и перезапустите бота.

---

**Примечание:** Если бот не является администратором группы, некоторые функции (автоматическое удаление пользователей, одобрение заявок) не будут работать. Это нормально для тестовой группы.
