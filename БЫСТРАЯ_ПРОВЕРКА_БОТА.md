# ⚡ Быстрая проверка бота на сервере

**Дата:** 02.12.2025

---

## 🔍 Шаг 1: Подключитесь к серверу

```bash
ssh root@81.200.146.32
```

---

## 📊 Шаг 2: Проверьте статус бота

```bash
systemctl status tradetherapybot.service
```

**Что искать:**

✅ **Если видите `active (running)`** - бот работает!
```
● tradetherapybot.service - Trade Therapy Bot
   Loaded: loaded
   Active: active (running) since ...
```

❌ **Если видите `inactive (dead)` или `failed`** - бот не работает!

---

## 📋 Шаг 3: Если бот не работает - посмотрите логи

```bash
journalctl -u tradetherapybot.service -n 30 --no-pager
```

Это покажет последние 30 строк логов. Ищите ошибки (ERROR, CRITICAL).

---

## 🔄 Шаг 4: Попробуйте перезапустить

```bash
sudo systemctl restart tradetherapybot.service
```

Подождите 3 секунды и снова проверьте:

```bash
systemctl status tradetherapybot.service
```

---

## 🐛 Частые проблемы и решения

### Проблема 1: Бот не запускается из-за ошибки в коде

**Решение:**
1. Посмотрите логи (шаг 3)
2. Найдите ошибку
3. Исправьте в коде локально
4. Сделайте `git push origin main`
5. Деплой обновит код автоматически

---

### Проблема 2: Отсутствует файл env_vars.txt

**Решение:**
```bash
cd /opt/tradetherapybot
nano env_vars.txt
```

Добавьте (если файл пустой):
```
BOT_TOKEN=ваш_токен_бота
ADMIN_ID=ваш_telegram_id
GROUP_CHAT_ID=id_группы
```

Сохраните: `Ctrl+O`, `Enter`, `Ctrl+X`

Перезапустите:
```bash
sudo systemctl restart tradetherapybot.service
```

---

### Проблема 3: База данных повреждена

**Решение:**
```bash
cd /opt/tradetherapybot

# Проверить бэкапы
ls -lh backups/

# Если есть бэкап - восстановить
cp backups/bot_backup_YYYYMMDD_HHMMSS.db bot.db

# Перезапустить
sudo systemctl restart tradetherapybot.service
```
