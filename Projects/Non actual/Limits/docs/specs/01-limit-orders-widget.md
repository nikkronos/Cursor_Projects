# Спецификация: Виджет пачки лимитных заявок (Лимиты)

## Что надо сделать
Виджет для выставления пачки лимитных заявок через API Т-Инвестиции: выбор инструмента (акции, фьючерсы), ввод суммы в рублях и % шага, одна кнопка — заявки на покупку ниже текущей цены, вторая — на продажу выше. Текущая цена обновляется раз в 10 секунд.

## Как это должно работать
- Пользователь открывает виджет → видит поиск инструмента, поле текущей цены (обновление раз в 10 сек), избранные (фильтр из localStorage).
- Выбирает инструмент → подставляется текущая цена (best bid/ask или last).
- Вводит: сумма в рублях (например 200 000), количество заявок (например 4), шаг в % (например 2).
- Зелёная кнопка → выставить N лимитных заявок на покупку (bid): цены на 2%, 4%, 6%, 8% ниже текущей; сумма делится поровну на N заявок; рубли переводятся в лоты с округлением вниз.
- Красная кнопка → выставить N лимитных заявок на продажу (ask): цены на 2%, 4%, 6%, 8% выше текущей; аналогично.
- Счёт: всегда первый из GetAccounts (один счёт у пользователя).
- Без диалога подтверждения перед выставлением.

## Инструменты
- Flask (backend), как в Damir.
- T-Invest REST API: UsersService/GetAccounts, InstrumentsService (Shares, Futures), MarketDataService/GetOrderBook, OrdersService/PostOrder.
- Frontend: HTML/CSS/JS, localStorage для избранного.

## Критерии приемки
- [ ] Поиск/выбор любого инструмента (акция, фьючерс) из списка API.
- [ ] Фильтр избранных (сохранение в localStorage).
- [ ] Отображение текущей цены, обновление раз в 10 сек.
- [ ] Поля: сумма (руб), кол-во заявок, шаг (%).
- [ ] Зелёная кнопка: лимитные покупки ниже текущей цены (шаг 2% → уровни -2%, -4%, -6%, -8%).
- [ ] Красная кнопка: лимитные продажи выше текущей цены (шаг 2% → уровни +2%, +4%, +6%, +8%).
- [ ] Сумма в рублях переводится в лоты автоматически (округление вниз), поровну на N заявок.
- [ ] Деплой на Timeweb 24/7 (инструкция в README).

## Риски и митигация
- **Лимит API 150 PostOrder/мин:** пауза между запросами при выставлении пачки (например 0.5–1 сек).
- **Цена из стакана:** используем lastPrice или mid (best_bid+best_ask)/2; при пустом стакане — ошибка/подсказка.
