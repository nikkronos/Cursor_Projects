# Архитектура — Лимиты

## Общая схема
- Браузер → Flask (сервер) → T-Invest REST API. Токен хранится на сервере (env).
- Один счёт: первый из GetAccounts.

## Эндпоинты backend
- `GET /` — отдача статики (виджет).
- `GET /api/accounts` — первый счёт (для подстановки accountId).
- `GET /api/instruments` — список инструментов (акции + фьючерсы) для поиска/выбора.
- `GET /api/orderbook?instrument_id=...` — стакан, текущая цена (кэш 10 сек на сервере по возможности).
- `POST /api/orders/limit-batch` — тело: instrument_id, direction (buy/sell), total_rub, count, step_pct. Ответ: результат по каждой заявке или ошибка.

## Frontend
- Один экран: поиск инструмента, избранное (localStorage), текущая цена, поля сумма/кол-во/шаг %, две кнопки (покупка ниже / продажа выше).
- Таймер обновления цены: 10 секунд.

## Безопасность
- Токен только в env на сервере. В браузер не передаётся.
