# Быстрый старт: HeadHunter Automation

## Шаг 1: Установка зависимостей

```powershell
cd HeadHunterAutomation
pip install -r requirements.txt
playwright install chromium
```

## Шаг 2: Настройка env_vars.txt

Откройте `env_vars.txt` в корне проекта и добавьте (или проверьте наличие):

```
HH_COVER_LETTER=Доброго времени суток.\n\nПоследние два года я развиваю собственные онлайн-проекты: платное Telegram-комьюнити трейдеров и подписочного бота с доходом до 130–140 тыс. рублей в месяц.\nДо этого 3 года работал в Т-Банке, дорос до руководителя группы 15–20 человек, отвечал за KPI, обучение и операционный порядок.\n\nПривык превращать хаос в систему: цели >> задачи >> дедлайны >> отчётность.\nЕжедневно работаю с Google Sheets/Docs, календарями, задачами, сейчас активно использую AI (Cursor, GPT) для автоматизации рутины.\n\nВсе остальные подробности в резюме.
```

**Важно:** Используйте `\n` для переносов строк в одной строке.

## Шаг 3: Авторизация на hh.ru

1. Откройте браузер
2. Залогиньтесь на https://spb.hh.ru
3. Оставьте браузер открытым (скрипт откроет свой браузер, но сессия может быть нужна)

## Шаг 4: Запуск

**Вариант 1: Через bat-файл (рекомендуется)**
```powershell
.\run_apply.bat
```

**Вариант 2: Напрямую через Python**
```powershell
python scripts\apply_to_vacancies.py
```

## Что происходит

1. Открывается браузер (не в headless режиме)
2. Переход на страницу поиска "project assistant"
3. Поиск и обработка до 10 вакансий
4. Для каждой вакансии:
   - Проверка, не откликались ли уже
   - Клик на "Откликнуться"
   - Проверка на тест (если есть - пропуск)
   - Заполнение сопроводительного письма
   - Подтверждение отклика
5. Сохранение истории в `data/applied_vacancies.json`
6. Логирование в `logs/`

## Повторный запуск

После обновления страницы на hh.ru можно запустить скрипт снова. Он автоматически пропустит уже откликнутые вакансии.

## Настройка количества вакансий

Откройте `scripts/apply_to_vacancies.py` и измените:
```python
MAX_VACANCIES = 10  # Измените на нужное количество (например, 30)
```

## Логи

Все действия логируются в:
```
logs/hh_automation_YYYYMMDD_HHMMSS.log
```

## История откликов

Сохранена в:
```
data/applied_vacancies.json
```

---

**Примечание:** Скрипт работает с видимым браузером для контроля процесса. После завершения браузер остается открытым на 10 секунд.
