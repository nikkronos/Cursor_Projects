# Инструкция по использованию скриптов автоматизации Google Forms

## Быстрый старт

### 1. Анализ структуры формы (опционально)

Перед первым запуском рекомендуется проанализировать структуру формы:

**Вариант 1: Через bat-файл (рекомендуется для Windows)**
```bash
# В командной строке (CMD):
run_analyze_form.bat

# В PowerShell:
.\run_analyze_form.bat

# Скрипт предложит выбрать режим:
# 1. Автоматический - попытка автоматического заполнения
# 2. Ручной - вы вручную переходите между страницами
```

**Вариант 2: Через командную строку**
```bash
cd ParentChildResearch

# Автоматический режим (по умолчанию)
python scripts\analyze_form_structure.py

# Ручной режим (если автоматический не работает)
python scripts\analyze_form_structure.py --manual
```

**Примечание**: Если скрипт не может автоматически заполнить первую страницу, он автоматически переключится в ручной режим и будет ждать, пока вы вручную перейдете между страницами.

Этот скрипт:
- Откроет Google Form в браузере
- Сохранит HTML структуру каждой страницы
- Проанализирует паттерны организации вопросов
- Сохранит результаты в `data/raw/form_analysis/`

### 2. Тестовый запуск

Для проверки работы скрипта на одной записи:

**Вариант 1: Через bat-файл (рекомендуется для Windows)**
```bash
# Двойной клик на файл или из командной строки:
run_submit_form.bat
```

**Вариант 2: Через командную строку**
```bash
cd ParentChildResearch
python scripts\submit_to_google_form.py
```

По умолчанию скрипт работает в тестовом режиме (`test_mode=True`), обрабатывая только первую запись.

### 3. Полный запуск

Для обработки всех записей из CSV файла:

Откройте `scripts/submit_to_google_form.py` и измените:
```python
test_mode = False  # Изменить с True на False
```

Затем запустите:
```bash
python scripts/submit_to_google_form.py
```

## Структура данных

### CSV файл должен содержать колонки:
- `Имя` - Фамилия и имя
- `Возраст` - Возраст
- `Класс` - Класс
- `Q1` - `Q60` - Ответы на вопросы 1-60 (значения 1-5)
- `Choice1` - `Choice42` - Выборы для вопросов 1-42 (значения 1 или 2)
- `ProfQ1` - `ProfQ20` - Ответы на вопросы о профессиях 1-20 (значения 1-5)

## Логи и прогресс

### Логи
Логи сохраняются в `logs/form_submission_YYYYMMDD_HHMMSS.log`

### Прогресс
Прогресс сохраняется в `data/generated/submission_progress.json`

Файл содержит:
- Индекс последней обработанной записи
- Количество успешных и неудачных отправок
- Детальную информацию о каждой записи

### Скриншоты при ошибках
Скриншоты сохраняются в `data/generated/screenshots/` при возникновении ошибок

## Стратегии поиска радиокнопок

Скрипт использует 4 стратегии поиска радиокнопок (в порядке приоритета):

1. **Поиск через listitem контейнеры** - находит контейнер вопроса и радиокнопки внутри него
2. **Группировка по вопросам** - группирует все радиокнопки по вопросам (5 для scale, 2 для choice)
3. **Поиск по data-value с позицией** - находит радиокнопки по значению и выбирает ближайшую
4. **XPath поиск** - использует XPath для поиска внутри контейнера вопроса

Если одна стратегия не работает, автоматически пробуется следующая.

## Устранение проблем

### Скрипт не находит радиокнопки
1. Запустите `analyze_form_structure.py` для анализа DOM
2. Проверьте логи в `logs/` для детальной информации
3. Убедитесь, что форма открывается корректно

### Ошибки при заполнении
1. Проверьте скриншоты в `data/generated/screenshots/`
2. Проверьте логи для детальной информации об ошибке
3. Убедитесь, что данные в CSV корректны

### Возобновление работы
1. Откройте `data/generated/submission_progress.json`
2. Найдите `last_processed_index`
3. Установите `start_index = last_processed_index + 1` в скрипте
