# Инструкция по работе с Version5

## Структура проекта

Все файлы находятся в папке `ParentChildResearch/Version5/`:
- `ВКР_ПОЛНЫЙ_ДОКУМЕНТ_v7.docx` - исходный документ (будет скопирован автоматически)
- `СПИСОК_ЛИТЕРАТУРЫ_ФИНАЛЬНЫЙ.md` - список литературы (будет скопирован автоматически)
- `analyze_v7.py` - скрипт для анализа документа
- `fix_v7_to_v8.py` - скрипт для исправления и создания v8
- `quick_test.py` - быстрый тест для проверки файлов

## Шаг 1: Подготовка

Откройте терминал (PowerShell или CMD) и перейдите в папку Version5:

```powershell
cd "C:\Users\krono\OneDrive\Рабочий стол\Cursor_test\ParentChildResearch\Version5"
```

Или просто дважды кликните на `START_ANALYSIS.bat`

## Шаг 2: Анализ документа

Запустите анализ:

```bash
python analyze_v7.py
```

Или используйте bat-файл:
```bash
START_ANALYSIS.bat
```

Скрипт:
1. Скопирует файлы из Version4 если их нет
2. Загрузит документ v7
3. Найдет все ссылки на источники
4. Проверит соответствие списку литературы
5. Найдет проблемные места (фамилии в скобках, неверные номера)
6. Сохранит результаты в `ANALYSIS_RESULTS.txt`

## Шаг 3: Исправление документа

После анализа запустите исправление:

```bash
python fix_v7_to_v8.py
```

Скрипт:
1. Исправит все проблемные ссылки
2. Уберет фамилии из скобок на странице 4
3. Перефразирует текст с правильным цитированием
4. Проверит все номера источников
5. Создаст документ `ВКР_ПОЛНЫЙ_ДОКУМЕНТ_v8.docx`

## Что исправляется

### 1. Страница 4 - фамилии в скобках

**Было:**
- "системный семейный подход (Андреева 5)"
- "(Эдельмин, Юскис 64)"

**Стало:**
- "системный семейный подход, который изучал Андреева [5]"
- "Эдельмин, Юскис [64]"

### 2. Все ссылки в документе

**Было:**
- "(Андреева 5)" → "Андреева [5]"
- "(Эдельмин, Юскис 64)" → "Эдельмин, Юскис [64]"

### 3. Проверка номеров источников

- Удаляются ссылки на источники, которых нет в списке литературы (номера > 53)
- Проверяется соответствие всех ссылок списку литературы

## Результаты

После выполнения вы получите:
- `ANALYSIS_RESULTS.txt` - подробный отчет об анализе
- `ВКР_ПОЛНЫЙ_ДОКУМЕНТ_v8.docx` - исправленный документ

## Устранение проблем

Если возникают ошибки:

1. **"python-docx not installed"**
   ```bash
   pip install python-docx
   ```

2. **"File not found"**
   - Убедитесь, что файлы скопированы из Version4
   - Запустите `python quick_test.py` для проверки

3. **Проблемы с кодировкой**
   - Убедитесь, что терминал поддерживает UTF-8
   - В PowerShell: `chcp 65001`
