# Интеграция N8N

## Общая информация

**N8N** - это платформа автоматизации рабочих процессов с открытым исходным кодом, позволяющая создавать сложные процессы, объединяя различные API и сервисы.

**Статус:** 🔴 В процессе настройки  
**Триал доступ:** 14 дней (уже получен)  
**Приоритет:** Высокий (1)

## Цели интеграции

1. Синхронизировать и настроить N8N
2. Интегрировать с существующими проектами (TradeTherapyBot, PastuhiBot)
3. Создать базовые workflow для автоматизации
4. Использовать N8N вкупе с Cursor для упрощения разработки

## Варианты установки N8N

### Вариант 1: N8N Cloud (рекомендуется для начала)

- Уже зарегистрирован аккаунт
- Триал доступ на 14 дней
- Не требует установки на сервер
- Доступ через веб-интерфейс

**Ссылка:** https://n8n.io/

### Вариант 2: Self-hosted (для продакшена)

- Установка на сервер Timeweb
- Полный контроль над данными
- Бесплатно (open-source)
- Требует настройки Docker или прямую установку

## Настройка подключения

### 1. Получение API ключа

1. Войдите в N8N Cloud (или self-hosted)
2. Перейдите в Settings → API
3. Создайте новый API ключ
4. Сохраните ключ в `env_vars.txt` (НЕ коммитить!)

### 2. Структура env_vars.txt для N8N

```
# N8N Configuration
N8N_API_URL=https://your-instance.n8n.cloud
N8N_API_KEY=your_api_key_here
N8N_WEBHOOK_URL=https://your-instance.n8n.cloud/webhook
```

## Интеграция с Cursor

### Настройка MCP-сервера для N8N

1. В Cursor перейдите в Settings → Features → MCP Servers
2. Нажмите "Add Server"
3. Введите параметры:
   - **Command:** `npx`
   - **Arguments:** `["-y", "n8n-workflow-builder-mcp"]`
   - **Environment Variables:**
     - `N8N_API_URL`: URL вашего N8N инстанса
     - `N8N_API_KEY`: ваш API ключ
     - `N8N_VERSION`: `1.72.1` (опционально)
4. Включите сервер и перезапустите Cursor
