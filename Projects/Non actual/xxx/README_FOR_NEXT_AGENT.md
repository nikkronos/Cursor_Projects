## Важные правила

### ⚠️ КРИТИЧЕСКИ ВАЖНО:

1. **Проект коммитится в Git** - все изменения через Git (в отдельную папку)
2. **Структура проекта:** По аналогии с PastuhiBot - юзербот (Pyrogram) + бот для управления (aiogram/pyTelegramBotAPI)
3. **Начало работы:** Сначала тестирование на тестовом канале
4. **Сервер:** Используется тот же сервер Timeweb, что и для TradeTherapyBot и PastuhiBot

### Работа с кодом:

- Используй `validators.py` для валидации входных данных (если будет создан)
- Логируй все важные операции через `logger`
- Обрабатывай все ошибки через `try-except`
- Используй параметризованные SQL-запросы (никогда f-строки!)
- Следуй паттернам из `../docs/patterns.md`

### Работа с Git:

- **Коммиты:** Проект коммитится в Git (в отдельную папку)
- **Перед коммитом:** Проверь, что все работает, нет ошибок линтера
