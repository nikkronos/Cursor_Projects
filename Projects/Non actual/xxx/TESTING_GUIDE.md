# Руководство по тестированию (Этап 6)

Тестирование копирования сообщений на тестовых каналах **до** перехода на реальный канал.

---

## 1. Подготовка тестового окружения

### 1.1 Создать тестовые каналы в Telegram

1. **Оригинальный канал (источник):**
   - Создайте канал (например: «Тест источник»).
   - Тип: публичный или приватный — по желанию.
   - Запомните ID канала (см. ниже).

2. **Целевой канал (куда копируем):**
   - Создайте второй канал (например: «Тест копия»).
   - Тип: публичный или приватный.
   - Запомните ID канала.

**Как узнать ID канала:**
- Добавьте в канал бота [@userinfobot](https://t.me/userinfobot) или [@getidsbot](https://t.me/getidsbot) и перешлите туда любое сообщение из канала — бот покажет `chat_id` (например: `-1001234567890`).
- Либо: сделайте канал публичным, откройте в браузере `https://t.me/username_канала` — иногда ID можно получить через API/боты по username.

### 1.2 Бот и юзербот

1. **Бот (Bot API):**
   - Создайте бота через [@BotFather](https://t.me/BotFather), получите `BOT_TOKEN`.
   - Добавьте этого бота **администратором** в **целевой** канал (права: публикация сообщений и при необходимости закрепление).

2. **Юзербот (аккаунт пользователя):**
   - Получите [API ID и API Hash](https://my.telegram.org/apps) для приложения.
   - Номер телефона — аккаунт, который будет «юзерботом».
   - Добавьте этот аккаунт **участником** (или админом) **оригинального** канала, чтобы он видел сообщения.

---

## 2. Настройка env_vars.txt

Скопируйте `env_vars.example.txt` в `env_vars.txt` и заполните **тестовыми** данными:

```text
BOT_TOKEN=123456:ABC-DEF...
ADMIN_ID=ваш_telegram_id

API_ID=12345678
API_HASH=abcdef1234567890...
PHONE_NUMBER=+79001234567

SOURCE_CHANNEL_ID=-1001234567890
TARGET_CHANNEL_ID=-1009876543210
```

- `SOURCE_CHANNEL_ID` — ID **оригинального** (тестового) канала.
- `TARGET_CHANNEL_ID` — ID **целевого** (тестового) канала.
- `ADMIN_ID` — ваш Telegram user ID (для возможных админ-функций).

Файл `env_vars.txt` в репозиторий не коммитится (указан в `.gitignore`).

---

## 3. Запуск приложения

1. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

2. Запуск из папки `xxx`:
   ```bash
   cd xxx
   python main.py
   ```

3. При первом запуске с Pyrogram появится запрос кода из Telegram и, при необходимости, двухфакторной аутентификации — введите код в консоль.

4. Ожидаемые логи:
   - `Bot instance created successfully.`
   - `Copy handler set for userbot.`
   - `Pyrogram userbot started successfully.`
   - `Access to source channel verified: ...`
   - `Starting management bot...`

Ошибки подключения юзербота или бота смотрите в `logs/userbot.log` и `logs/bot.log`.

---

## 4. Проверка копирования по типам сообщений

В **оригинальный** (тестовый) канал отправьте по очереди:

| № | Тип сообщения        | Действие                         | Проверить в целевом канале      |
|---|----------------------|----------------------------------|----------------------------------|
| 1 | Текст                | Обычное текстовое сообщение     | Появился такой же текст         |
| 2 | Фото с подписью      | Фото + подпись                   | Фото и подпись скопированы      |
| 3 | Видео с подписью     | Видео + подпись                  | Видео и подпись скопированы     |
| 4 | Документ             | Файл (документ)                  | Документ скопирован             |
| 5 | Голосовое сообщение | Голосовое сообщение             | Голосовое скопировано           |
| 6 | Стикер               | Стикер                           | Стикер скопирован               |
| 7 | GIF (анимация)       | GIF                              | Анимация скопирована            |
| 8 | Альбом (несколько фото) | 2–3 фото в одном сообщении   | Все фото появились (по одному или группой) |

После каждого типа проверьте:
- сообщение появилось в **целевом** канале;
- в `logs/bot.log` нет критичных ошибок;
- при необходимости — запись в БД в таблице `messages` (если смотрите через SQLite).

---

## 5. Проверка обработки ошибок

1. **Нет доступа к каналу:**
   - Временно укажите неверный `SOURCE_CHANNEL_ID` или уберите юзербота из канала — в логах должна появиться ошибка/предупреждение, приложение не должно падать.

2. **Бот не админ в целевом канале:**
   - Снимите бота с прав администратора в целевом канале и отправьте сообщение в источник — в логах должна быть ошибка отправки в канал, копирование не выполнится.

3. **Остановка приложения:**
   - Остановите приложение через Ctrl+C — в логах должно быть корректное завершение (остановка юзербота и бота).

---

## 6. Логи и база данных

- **Логи:**
  - `logs/bot.log` — бот (aiogram), копирование, общие ошибки.
  - `logs/userbot.log` — юзербот (Pyrogram), приём сообщений из источника.

- **База данных:**
  - Файл `bot.db` (SQLite) — таблица `messages` хранит факт копирования (message_id, source/target channel, type, copied_at).
  - Опционально можно смотреть через: `sqlite3 bot.db "SELECT * FROM messages;"`

---

## 7. Чек-лист перед переходом на реальный канал

- [ ] Все проверенные типы сообщений копируются в тестовый целевой канал.
- [ ] В логах нет критичных ошибок при нормальной работе.
- [ ] Ошибки (нет доступа, нет прав бота) обрабатываются и пишутся в логи.
- [ ] Остановка по Ctrl+C проходит без падения.
- [ ] Решено, как будете хранить и использовать реальные `SOURCE_CHANNEL_ID` и `TARGET_CHANNEL_ID` на сервере (env_vars или секреты).

После прохождения чек-листа можно переходить к **Этапу 7** (релиз на реальный канал): подставить реальные каналы в `env_vars.txt` и запустить на сервере Timeweb.
