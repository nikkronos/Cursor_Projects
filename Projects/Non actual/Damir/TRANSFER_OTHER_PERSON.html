# Damir — Futures Auction Widget (T-Invest) | Инструкция для передачи другому человеку

## Коротко: что это
Виджет на `Flask` + `static/index.html` для отображения таблицы инструментов T-Invest (фьючерсы/спот) с автообновлением и “именными ссылками” через `/?profile=...`.

Текущая рабочая версия интерфейса: **v5**.

## Важно про актуальность README на GitHub
`README.md` в репозитории местами описывает старую логику UI (про “Свечи/Стакан” и другие интервалы), поэтому при правках ориентироваться нужно на фактический код:
- `server.py`
- `static/index.html`

## Как открыть виджет (именые ссылки / profile)
Важно: корень `/` **без** `profile` отдаёт заглушку (виджет и настройки не грузятся). Нужен URL с параметром.

Пример формата:
- http://81.200.146.32:5000/?profile=ivan
- http://81.200.146.32:5000/?profile=damir
- http://81.200.146.32:5000/?profile=nikolay
- http://81.200.146.32:5000/?profile=alex
- http://81.200.146.32:5000/?profile=nikita

## Настройки и перенос на другое устройство
Выбор инструментов хранится в `localStorage` браузера (на сервере профили пока не хранятся).
Это значит: на новом устройстве/браузере нужно будет заново открыть настройки и выбрать инструменты.

## Timeweb / деплой (Linux + systemd). Ориентир: /opt/futures_auction
### Что должно быть в папке `/opt/futures_auction`
- `server.py`
- `static/index.html`
- `requirements.txt` (может быть, если делаем установки на месте)
- `env_vars.txt` (секреты) — **НЕ коммитить**
- виртуальное окружение: `/opt/futures_auction/.venv/` (создаётся при настройке)

### env_vars.txt
Обязательно задать:
- `TINKOFF_INVEST_TOKEN=...`
- `SANDBOX=0` (всегда боевой контур)

Пример:
- https://raw.githubusercontent.com/nikkronos/Futures_auction/main/env_vars.example.txt

### Вариант “копируются файлы” (самый вероятный для Timeweb)
1. Скопировать на сервер в `/opt/futures_auction/`:
   - `server.py`
   - `static/index.html`
   - `requirements.txt` (если нужно)
2. Создать/обновить venv:
   - `python3 -m venv /opt/futures_auction/.venv`
   - `/opt/futures_auction/.venv/bin/pip install -r /opt/futures_auction/requirements.txt`
3. Положить секреты в:
   - `/opt/futures_auction/env_vars.txt`
4. Поднять systemd сервис:
   - `/etc/systemd/system/futures_auction.service`
   - `sudo systemctl daemon-reload`
   - `sudo systemctl enable futures_auction`
   - `sudo systemctl restart futures_auction`
   - `sudo systemctl status futures_auction --no-pager`

### Вариант “через git” (если git доступен на сервере)
- В папке `/opt/futures_auction`:
  - `git pull`
  - затем `pip install -r requirements.txt`
  - затем `systemctl restart futures_auction`

## Проверка после деплоя (минимум)
1. Открыть заглушку без profile:
   - http://81.200.146.32:5000/
   Должна быть страница “Доступ по личным ссылкам…”.
2. Открыть виджет с profile:
   - http://81.200.146.32:5000/?profile=nikita
3. UI:
   - кнопка `Настройки` открывает/закрывает панель
   - `Авто: ВКЛ/ВЫКЛ` работает
   - таблица не “пропадает” при ошибках/таймаутах
4. Поведение при лимите инструментов:
   - максимум обновления/кэша: `40`

## Критические правила поведения при правках (важно для Cursor/Claude)
При любых изменениях сохранять текущее поведение v5:
1. Корень `/` **без** `?profile=...`:
   - не делать запросы к API
   - отдавать заглушку
2. Ошибки/таймауты `/api/orderbook`:
   - таблицу не очищать
   - оставлять последние валидные данные
3. Лимит инструментов:
   - в коде/фронте жестко согласован лимит `40`
4. Автообновление и расписание аукциона:
   - расписание считается по МСК в двух местах:
     - `server.py` (логика `_is_auction_time()`)
     - `static/index.html` (логика `getRefreshInterval()`)
5. История “поломки кнопок” (двойные обработчики):
   - не добавлять дублирующие `addEventListener` к кнопкам “Настройки/Авто”
   - если меняете обработчики — убедиться, что срабатывает один раз и ошибки из try/catch не ломают UI

API endpoints:
- `/api/futures` — список инструментов для настроек
- `/api/orderbook` — данные таблицы + auction status + `limit_warning` (если превышен лимит)
- `/api/stats` — статистика запросов/кэша (если доступна)
