# Обновление кода на сервере

## Проблема: Заглушка не работает

Если заглушка не отображается, значит код не обновился на сервере.

## Решение: Обновить код на сервере

### Вариант 1: Через Git (если репозиторий на сервере)

```bash
ssh root@81.200.146.32
cd /opt/tradetherapybot
git pull origin main
systemctl restart tradetherapybot.service
systemctl status tradetherapybot.service
```

### Вариант 2: Через GitHub Desktop (автоматический деплой)

1. **Закоммитьте изменения локально:**
   ```bash
   git add handlers/user.py services.py database.py
   git commit -m "feat: добавлена заглушка тарифов до 25.12"
   git push origin main
   ```

2. **GitHub Actions автоматически задеплоит на сервер** (если настроен CI/CD)

3. **Проверьте статус деплоя:**
   - Откройте GitHub → Actions
   - Убедитесь, что деплой прошел успешно

4. **Проверьте бота:**
   - Отправьте `/start` боту
   - Нажмите "Тарифы"
   - Должна появиться заглушка: "Информация появится здесь 25 декабря."

### Вариант 3: Ручное копирование файлов

Если Git не настроен на сервере:

```bash
# На локальном компьютере
scp handlers/user.py root@81.200.146.32:/opt/tradetherapybot/handlers/
scp services.py root@81.200.146.32:/opt/tradetherapybot/
scp database.py root@81.200.146.32:/opt/tradetherapybot/

# На сервере
ssh root@81.200.146.32
cd /opt/tradetherapybot
systemctl restart tradetherapybot.service
systemctl status tradetherapybot.service
```

## Проверка работы заглушки

После обновления:

1. **Отправьте боту `/start`**
2. **Нажмите кнопку "Тарифы"**
3. **Должно появиться сообщение:**
   ```
   Тарифы
   
   Информация появится здесь 25 декабря.
   ```

Если заглушка не появилась:
- Проверьте логи: `journalctl -u tradetherapybot.service -n 50`
- Убедитесь, что код обновился: `cd /opt/tradetherapybot && git log -1`
