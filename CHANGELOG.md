# История изменений

## [2.0.0] - Миграция на SQLite и рефакторинг

### Добавлено
- Команда `/backup` для администратора (получение файла базы данных)
- Модульная архитектура: код разделен на `main.py`, `handlers.py`, `services.py`, `database.py`, `loader.py`
- Автоматическое создание базы данных SQLite при первом запуске

### Изменено
- **Миграция с PostgreSQL на SQLite** - больше не требуется отдельный сервер базы данных
- Обновлены все SQL-запросы с синтаксиса PostgreSQL (`%s`) на SQLite (`?`)
- Улучшена работа с соединениями БД (использование Context Manager для надежности)
- Обновлена документация под новую архитектуру

### Удалено
- Зависимость от `psycopg2-binary`
- Параметры подключения к PostgreSQL из конфигурации (`DB_NAME`, `DB_USER`, `DB_PASSWORD`, `DB_HOST`)
- Старый монолитный файл `bot.py` (заменен на модульную структуру)

### Технические детали
- Все функции работы с БД теперь используют `with get_db_connection() as conn:` для гарантированного закрытия соединений
- Даты хранятся как строки в SQLite и конвертируются через функции `parse_db_date()` и `format_db_date()`
- Файл базы данных `bot.db` добавлен в `.gitignore`

### Рекомендации
- Отключите услугу PostgreSQL на TimeWeb для экономии средств
- Регулярно делайте бэкап базы данных командой `/backup`
- При переносе на новый сервер просто скопируйте файл `bot.db` вместе с кодом
