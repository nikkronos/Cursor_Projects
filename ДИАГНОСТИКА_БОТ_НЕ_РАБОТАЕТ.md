# 🔧 Диагностика: Бот не работает

**Дата:** 02.12.2025

---

## ✅ Пошаговая проверка

### 1. Проверить статус на сервере

Подключитесь к серверу по SSH:

```bash
ssh -p 22 root@81.200.146.32
```

Проверьте статус сервиса:

```bash
systemctl status tradetherapybot.service
```

**Что искать:**
- ✅ `active (running)` - бот работает
- ❌ `inactive (dead)` - бот не запущен
- ❌ `failed` - бот упал с ошибкой

---

### 2. Посмотреть логи

Если бот не работает, смотрите логи:

```bash
# Последние 50 строк логов
journalctl -u tradetherapybot.service -n 50

# Логи в реальном времени
journalctl -u tradetherapybot.service -f
```

**Что искать:**
- ❌ Ошибки (ERROR, CRITICAL)
- ❌ Проблемы с токеном бота
- ❌ Проблемы с базой данных
- ❌ Проблемы с зависимостями

---

### 3. Попробовать перезапустить

```bash
sudo systemctl restart tradetherapybot.service
```

Подождите 5 секунд и проверьте статус:

```bash
systemctl status tradetherapybot.service
```

---

### 4. Проверить файлы на сервере

```bash
cd /opt/tradetherapybot

# Проверить наличие файлов
ls -la

# Проверить наличие env_vars.txt
cat env_vars.txt

# Проверить наличие bot.db
ls -lh bot.db
```

---

### 5. Проверить последний деплой

Откройте GitHub Actions:
1. https://github.com/nikkronos/TradeTherapyBot
2. Вкладка **Actions**
3. Найдите последний **Deploy to Server**
4. Проверьте, был ли он успешным

## 🐛 Частые проблемы

### Проблема 1: Бот упал после деплоя

**Симптомы:**
- В логах есть ошибки
- Service показывает `failed`

**Решение:**
1. Посмотрите логи (шаг 2 выше)
2. Исправьте ошибку в коде
3. Сделайте новый push
4. Деплой обновит код автоматически

---

### Проблема 2: Отсутствует env_vars.txt

**Симптомы:**
- Бот не может найти токен
- Ошибка: `BOT_TOKEN not found`

**Решение:**
```bash
cd /opt/tradetherapybot
nano env_vars.txt
```

Добавьте:
```
BOT_TOKEN=ваш_токен
ADMIN_ID=ваш_id
GROUP_CHAT_ID=id_группы
```

Сохраните (Ctrl+O, Enter, Ctrl+X) и перезапустите:

```bash
sudo systemctl restart tradetherapybot.service
```

---

### Проблема 3: База данных повреждена или отсутствует

**Симптомы:**
- Ошибки в логах про базу данных
- `bot.db` отсутствует или пустой

**Решение:**
```bash
cd /opt/tradetherapybot

# Проверить бэкапы
ls -lh backups/

# Восстановить из бэкапа (если есть)
cp backups/bot_backup_YYYYMMDD_HHMMSS.db bot.db

# Или создать новую (данные будут потеряны)
# Бот создаст новую автоматически при запуске
```

Перезапустите:
```bash
sudo systemctl restart tradetherapybot.service
```

---

### Проблема 4: Бот не отвечает на команды

**Симптомы:**
- Бот запущен (status: active)
- Но не отвечает на `/start`

**Возможные причины:**
1. Токен неверный
2. Бот заблокирован пользователем
3. Проблемы с сетью

**Решение:**
1. Проверьте токен в `env_vars.txt`
2. Попробуйте отправить `/start` с другого аккаунта
3. Проверьте логи на наличие ошибок
