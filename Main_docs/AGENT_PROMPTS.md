# Инструкция для AI-агентов (Claude Code)

> Этот файл — детализация к `PROFILE.md` и `CLAUDE.md`.
> Читай, если нужен расширенный контекст по процессу работы.

---

## Старт сессии

1. `git status` + `git log --oneline -3` — понять состояние репо
2. Читай `Main_docs/PROFILE.md` — кто владелец, принципы, техстек
3. Если есть `CLAUDE.md` в папке проекта — читай его (приоритет над этим файлом)
4. Читай `ROADMAP`, `SESSION_SUMMARY` проекта — понять текущий контекст

---

## Рабочий процесс

### Планирование
- Задай уточняющие вопросы до начала реализации, если задача неоднозначна
- Коротко обозначь план перед тем, как менять код
- При риске сломать существующую функциональность — предупреди

### Реализация
- Следуй паттернам проекта (смотри `docs/patterns.md` если есть)
- Параметризованные SQL-запросы — никогда не f-строки
- Логируй важные операции (INFO/WARNING/ERROR)
- Обрабатывай ошибки через `try/except`, давай понятные сообщения пользователю
- Не хардкоди секреты — читай из `env_vars.txt` / `.env` / `config.py`

### После изменений
- Проверить diff (`git diff`)
- Тест/линт если есть
- `git add` → `git commit` → `git push` (push обязателен)
- Не пушить заведомо сломанное без явной причины

---

## Завершение сессии

1. Обновить `SESSION_SUMMARY_ДАТА.md` — что сделано, открытые задачи, техдетали
2. Перенести закрытые задачи в `DONE_LIST`
3. Обновить `ROADMAP` — убрать выполненное, добавить новое если есть
4. Убедиться, что все изменения запушены

## Критические правила

- Радикальная честность: ошибки называть прямо, не льстить
- Компактные ответы — без мотивационных речей и generic advice
- Парето: сначала рабочие 80%, затем доводка. Не жертвовать безопасностью и деньгами
- Sonnet по умолчанию, Opus только если оправдано
- Оккам: не плодить файлы, фреймворки, сущности без необходимости
