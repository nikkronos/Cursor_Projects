# Быстрый старт для работы с AI агентом

## 🚀 НАЧАЛО РАБОТЫ

Скопируй и вставь в новый чат с агентом:

```
Прочитай Main_docs/RULES_CURSOR.md - общие правила работы с проектами.

Определи, с каким проектом работаем (VPN в `Projects/VPN/`; архив: TradeTherapyBot, PastuhiBot, Damir, xxx, Limits, HH, Chess — в `Projects/Non actual/ИМЯ/`).

Изучи проект в папке `Projects/Non actual/ИМЯПРОЕКТА/` (архив) или `Projects/VPN/` (активный VPN), либо другом актуальном пути из Main_docs/PROJECTS.md:
1. ROADMAP_ИМЯПРОЕКТА.md - что в планах
2. DONE_LIST_ИМЯПРОЕКТА.md - что уже сделано
3. SESSION_SUMMARY_ДАТА.md - последняя сессия

Прочитай базу знаний в папке …/docs/ этого проекта (для архива обычно `Projects/Non actual/ИМЯПРОЕКТА/docs/`).

Есть задача: [ОПИШИ СВОЮ ЗАДАЧУ]

Используй workflow:
1. Читай …/docs/agent-onboarding.md в каталоге проекта
2. Создай спек в …/docs/specs/ используя spec-template.md (шаблон можно взять из архива TradeTherapyBot: `Projects/Non actual/TradeTherapyBot/docs/specs/spec-template.md`)
3. Задай мне вопросы
4. Проведи самооценку решения (риски, альтернативы)
5. Создай чеклист для самопроверки
6. Реализуй с тестами
```

## ✅ КОНЕЦ РАБОТЫ

Скопируй и вставь когда задача выполнена:

```
Завершай задачу. Проведи финальную проверку:

1. Чеклист из спека выполнен?
2. **Проверены все связанные функции?** (используй grep/codebase_search)
3. **Все кнопки имеют обработчики?**
4. **Навигация между меню работает?**
5. Все тесты написаны? (предложи запустить test_manual.py)
6. Соответствует patterns.md?
7. Логирование добавлено?
8. Нет hardcoded секретов?
9. Документация обновлена?
10. Линтер не находит ошибок?
11. Временные файлы удалены?

**КРИТИЧЕСКИ ВАЖНО:** Перед коммитом убедись, что не сломались существующие функции!

Обнови документацию:
- SESSION_SUMMARY_ДАТА.md - текущая сессия (только эта сессия!)
- DONE_LIST_ИМЯПРОЕКТА.md - перенеси старую информацию из SESSION_SUMMARY, добавь новые задачи
- ROADMAP_ИМЯПРОЕКТА.md - отметь выполненные задачи

Подготовь для Git (если проект коммитится):
- git add [какие файлы]
- git commit -m "[что сделано]"
- git push
```

## 📖 Где что находится:

### Общие правила:
- **`Main_docs/RULES_CURSOR.md`** - общие правила для всех проектов (ОБЯЗАТЕЛЬНО ПРОЧИТАТЬ!)

### В каждом проекте (Projects/ИМЯПРОЕКТА/):
- **`ROADMAP_ИМЯПРОЕКТА.md`** - планы развития проекта
- **`DONE_LIST_ИМЯПРОЕКТА.md`** - история выполненных задач
- **`SESSION_SUMMARY_ДАТА.md`** - информация о последней сессии
- **`README_FOR_NEXT_AGENT.md`** - инструкция для следующего агента
- **`docs/`** - база знаний проекта
  - `agent-onboarding.md` - онбординг
  - `architecture.md` - архитектура
  - `patterns.md` - правила кода
  - `specs/spec-template.md` - шаблон спецификаций

### Полная инструкция:
- **`Main_docs/AGENT_PROMPTS.md`** - полная инструкция для агента (детальное руководство)

### Проект VPN (путь и репо):
- Папка в workspace: **`Projects/VPN/`** — полный путь Windows и ссылка на GitHub: **`Main_docs/PROJECTS.md`** (раздел «7. VPN»), в самом проекте см. **`Projects/VPN/README_FOR_NEXT_AGENT.md`**
