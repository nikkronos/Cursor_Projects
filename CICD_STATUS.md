## ⏳ Что нужно сделать для запуска

### Шаг 1: Создать SSH ключ
```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\setup-ssh-key.ps1
```

### Шаг 2: Добавить публичный ключ на сервер
См. инструкцию в `QUICK_START_CICD.md`

### Шаг 3: Настроить GitHub Secrets
Нужно добавить 6 секретов в GitHub:
- `SSH_PRIVATE_KEY`
- `SSH_HOST` = `81.200.146.32`
- `SSH_PORT` = `22`
- `SSH_USER` = `root`
- `DEPLOY_PATH` = `/opt/tradetherapybot`
- `SERVICE_NAME` = `tradetherapybot`

### Шаг 4: Протестировать
Сделать тестовый коммит и проверить работу в GitHub Actions

---

## 📋 Как это работает

1. **Push в ветку `main`** → автоматически запускаются тесты
2. **Тесты прошли** → автоматически запускается деплой
3. **Деплой:**
   - Создает бэкап БД
   - Обновляет код через `git pull`
   - Обновляет зависимости
   - Перезапускает systemd service
   - Проверяет статус
   - При ошибке → автоматический откат

---

## 🔍 Проверка готовности

После настройки SSH и GitHub Secrets:
1. Сделайте тестовый коммит
2. Откройте GitHub → Actions
3. Убедитесь, что оба workflow выполнились успешно

## ⚠️ Важные замечания

1. **SSH ключ** должен быть создан один раз и добавлен в GitHub Secrets
2. **Публичный ключ** должен быть добавлен на сервер в `~/.ssh/authorized_keys`
3. **GitHub Secrets** настроены один раз и используются автоматически
4. После настройки всё работает автоматически при каждом `git push`

---

## 🆘 Если что-то не работает

1. Проверьте SSH подключение: `scripts/test-ssh-connection.ps1`
2. Проверьте логи в GitHub Actions
3. См. раздел "Устранение проблем" в `docs/CICD_SETUP.md`
4. Используйте ручной деплой (инструкция в `docs/CICD_SETUP.md`)
