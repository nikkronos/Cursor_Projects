# Финальная настройка Git на сервере

## Что нужно сделать сейчас

### Шаг 1: Подключиться к серверу

```powershell
ssh root@81.200.146.32
```

### Шаг 2: Перейти в папку проекта

```bash
cd /opt/tradetherapybot
```

### Шаг 3: Проверить бэкапы базы данных

```bash
ls -la backups/
```

Если там есть файлы бэкапов, восстановите последний:

```bash
ls -lt backups/ | head -5
```

Найдите самый свежий файл (например, `bot_backup_20251202_215700.db`) и восстановите:

```bash
cp backups/ИМЯ_ФАЙЛА.db bot.db
```

### Шаг 4: Переключиться на ветку main

```bash
git checkout -b main origin/main
```

Или если команда не работает:

```bash
git branch -M main
```

### Шаг 5: Проверить статус

```bash
git status
```

Должно показать: `On branch main` и `Your branch is up to date with 'origin/main'`

### Шаг 6: Выйти

```bash
exit
```

---

## Если базы данных нет в backups/

### Вариант 1: Получить через команду бота

1. Если бот еще работает, напишите ему в Telegram команду `/backup`
2. Бот отправит вам файл базы данных
3. Загрузите его на сервер

### Вариант 2: Создать новую базу

Если бэкапа нет, бот создаст новую базу данных при следующем запуске, но все данные будут потеряны.

---

## После этого

Когда git настроен и база данных восстановлена, деплой должен работать. Попробуйте сделать тестовый коммит.
