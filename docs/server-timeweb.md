# Документация о сервере Timeweb

Этот документ содержит информацию о сервере Timeweb для понимания AI агентами структуры и логики работы.

## Общая информация

- **Провайдер:** Timeweb
- **Статус:** Оплаченный сервер, работает
- **Проекты на сервере:**
  - TradeTherapyBot (создан с AI агентами)
  - PastuhiBot (создан с разработчиком, копия с сервера)
  - VPN (WireGuard-нода для тестирования, временно на этом сервере)

## Безопасность

### Хранение секретов

**ВАЖНО:** Все секреты (пароли, токены, API ключи) хранятся в файлах окружения:

- **Локально:** `env_vars.txt` (в корне каждого проекта)
- **На сервере:** `.env` или `env_vars.txt` (в директории проекта)

**Правила:**
- Файлы с секретами **НИКОГДА** не коммитятся в Git
- Используется `.gitignore` для исключения этих файлов
- Для примера структуры используется `env_vars.example.txt` (без реальных значений)

### Структура env_vars.txt

Пример структуры (без реальных значений):
```
BOT_TOKEN=your_bot_token_here
ADMIN_ID=your_telegram_id
GROUP_CHAT_ID=your_group_chat_id
API_ID=your_api_id
API_HASH=your_api_hash
```

## Подключение к серверу

### SSH подключение

- **IP адрес:** (хранится в env_vars.txt, не коммитится)
- **Пользователь:** root (или другой, хранится в env_vars.txt)
- **Пароль:** (хранится в env_vars.txt, не коммитится)

**Команда для подключения:**
```bash
ssh root@<IP_ADDRESS>
```

### Порты

- Закрытые порты: 25, 587, 389, 3389, 465, 2525, 53413
- Открытые порты: стандартные (22 для SSH, 80/443 для HTTP/HTTPS)

## Управление ботами на сервере

### TradeTherapyBot

**Команды для управления:**
```bash
# Проверить статус
sudo systemctl status tradetherapybot.service

# Остановить
sudo systemctl stop tradetherapybot.service

# Запустить
sudo systemctl start tradetherapybot.service

# Перезапустить
sudo systemctl restart tradetherapybot.service

# Посмотреть логи
sudo journalctl -u tradetherapybot.service -f
```

## Деплой

### Автоматический деплой (TradeTherapyBot)

- Настроен через GitHub Actions
- При `git push origin main` автоматически:
  1. Запускаются тесты
  2. При успешных тестах → деплой на сервер
  3. Автоматический перезапуск бота

### Ручной деплой

Если нужно обновить вручную:
1. Подключиться к серверу через SSH
2. Перейти в директорию проекта
3. Обновить код (git pull или копирование файлов)
4. Перезапустить сервис

## Резервное копирование

- База данных SQLite должна регулярно бэкапиться
- Для TradeTherapyBot есть команда `/backup` в боте
- Рекомендуется настроить автоматические бэкапы

## Мониторинг

- Логи через `journalctl` для systemd сервисов
- Рекомендуется настроить мониторинг ошибок
- Проверять статус сервисов регулярно

## Важные замечания для AI агентов

1. **Никогда не коммитить секреты** - все секреты в env_vars.txt
2. **Проверять .gitignore** - убедиться, что секретные файлы исключены
3. **Документировать изменения** - если меняется структура сервера, обновить эту документацию
4. **Безопасность превыше всего** - не хранить секреты в коде или документации
