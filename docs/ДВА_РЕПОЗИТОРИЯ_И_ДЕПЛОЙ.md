# Два репозитория и деплой: пошаговая инструкция

## Зачем два репозитория

- **nikkronos/TradeTherapyBot** — только проект TradeTherapyBot (бот для сообщества трейдеров). Коммиты и деплой относятся только к нему.
- **nikkronos/kopiya-iksuyemsya** (или другое имя) — только проект «Копия иксуюемся» (бот-перелив из канала в канал). Свой коммит и свой деплой.

В итоге: коммит в корень `Cursor_test` → TradeTherapyBot; коммит в папку `xxx` → Копия иксуюемся.

---

## Часть 1. Один раз: разделить репозитории

### Шаг 1.1. Создать новый репозиторий на GitHub для «Копия иксуюемся»

1. Зайти на https://github.com/new.
2. **Repository name:** например `kopiya-iksuyemsya` (или `xxx`, как удобнее).
3. **Description:** по желанию, например «Telegram-бот: копирование сообщений из канала в канал».
4. Выбрать **Public**.
5. **Не** ставить галочки «Add a README», «Add .gitignore» — репозиторий должен быть пустой.
6. Нажать **Create repository**.

После создания GitHub покажет URL, например:
`https://github.com/nikkronos/kopiya-iksuyemsya.git`

---

### Шаг 1.2. Убрать папку xxx из репозитория TradeTherapyBot

Выполнять **в корне** папки `Cursor_test` (это клон TradeTherapyBot):

```powershell
cd "c:\Users\krono\OneDrive\Рабочий стол\Cursor_test"

# Убрать папку xxx из индекса Git (папка останется на диске, но репозиторий её больше не отслеживает)
git rm -r --cached xxx

# Проверить, что в коммит попадёт только удаление xxx из репозитория
git status
```

Должно быть: `deleted: xxx/...` (много файлов). Папка `xxx` на диске при этом остаётся.

---

### Шаг 1.3. Обновить репозиторий TradeTherapyBot (gitignore и deploy)

В том же корне `Cursor_test` уже должны быть внесены такие изменения (если нет — внести вручную):

1. **.gitignore** — в конец добавить строку:
   ```
   xxx/
   ```
   Чтобы папка `xxx` никогда снова не попадала в коммиты TradeTherapyBot.

2. **.github/workflows/deploy.yml** — убрать блок, связанный с `xxx` (оставить только деплой TradeTherapyBot). В скрипте деплоя должны остаться только:
   - `cd ${{ secrets.DEPLOY_PATH }}`
   - `git pull origin main`
   - `pip3 install -r requirements.txt ...`
   - `sudo systemctl restart tradetherapybot.service`
   - проверка статуса `tradetherapybot.service`

После этого:

```powershell
git add .
git status
git commit -m "chore: Копия иксуюемся вынесена в отдельный репозиторий

- xxx/ удалён из репозитория TradeTherapyBot
- xxx/ добавлен в .gitignore
- deploy.yml только для TradeTherapyBot"
git push origin main
```

Теперь в **nikkronos/TradeTherapyBot** папки `xxx` нет, коммиты и деплой — только для TradeTherapyBot.

---

### Шаг 1.4. Сделать папку xxx отдельным репозиторием и отправить в новый репозиторий

Выполнять **в папке xxx**:

```powershell
cd "c:\Users\krono\OneDrive\Рабочий стол\Cursor_test\xxx"

# Инициализировать свой Git в папке xxx
git init

# Добавить все файлы (учитывается локальный .gitignore xxx — env_vars.txt, *.session и т.д.)
git add .

# Первый коммит
git commit -m "Initial commit: Копия иксуюемся (userbot + aiogram)"

# Имя ветки main (как на GitHub по умолчанию)
git branch -M main

# Подключить удалённый репозиторий (замените на свой URL из шага 1.1)
git remote add origin https://github.com/nikkronos/kopiya-iksuyemsya.git

# Отправить код в новый репозиторий
git push -u origin main
```

После этого проект «Копия иксуюемся» живёт в репозитории **nikkronos/kopiya-iksuyemsya**. Папка `xxx` на вашем ПК — это полноценный клон этого репозитория (со своим `.git`).

## Часть 3. Деплой на сервер (детально)

Сервер может быть один (например, Timeweb), но два бота разложены по разным папкам и разным systemd-сервисам.

### 3.1. TradeTherapyBot (репозиторий nikkronos/TradeTherapyBot)

1. **Клонирование (один раз):**
   ```bash
   sudo mkdir -p /opt/tradetherapybot
   sudo git clone https://github.com/nikkronos/TradeTherapyBot.git /opt/tradetherapybot
   ```

2. **На сервере:** создать `env_vars.txt` в `/opt/tradetherapybot`, заполнить переменные для TradeTherapyBot.

3. **Systemd:** скопировать и включить юнит:
   ```bash
   sudo cp /opt/tradetherapybot/tradetherapybot.service /etc/systemd/system/
   sudo nano /etc/systemd/system/tradetherapybot.service
   ```
   Указать: `WorkingDirectory=/opt/tradetherapybot`, `ExecStart=/usr/bin/python3 /opt/tradetherapybot/main.py`.

   ```bash
   sudo systemctl daemon-reload
   sudo systemctl enable tradetherapybot.service
   sudo systemctl start tradetherapybot.service
   ```

4. **GitHub Actions (CI/CD):** в репозитории TradeTherapyBot в **Settings → Secrets and variables → Actions** задать:
   - `SSH_HOST`, `SSH_USER`, `SSH_PRIVATE_KEY`, `SSH_PORT`
   - `DEPLOY_PATH=/opt/tradetherapybot`

   Тогда при `git push origin main` в TradeTherapyBot workflow сделает на сервере: `cd /opt/tradetherapybot`, `git pull`, `pip install -r requirements.txt`, `systemctl restart tradetherapybot.service`.

---

### 3.2. Копия иксуюемся (репозиторий nikkronos/kopiya-iksuyemsya)

1. **Клонирование (один раз):**
   ```bash
   sudo mkdir -p /opt/kopiya-iksuyemsya
   sudo git clone https://github.com/nikkronos/kopiya-iksuyemsya.git /opt/kopiya-iksuyemsya
   ```

2. **На сервере:** в `/opt/kopiya-iksuyemsya` создать `env_vars.txt` с переменными для бота (BOT_TOKEN, API_ID, API_HASH, PHONE_NUMBER, SOURCE_CHANNEL_ID, TARGET_CHANNEL_ID и т.д.).  
   **Первый запуск:** один раз зайти в папку и выполнить `python3 main.py`, чтобы Pyrogram создал сессию (ввод кода из Telegram). После успешной авторизации можно остановить бота и перейти на systemd.

3. **Systemd:** в репозитории kopiya-iksuyemsya есть файл `kopiya-iksuyemsya.service`. На сервере:
   ```bash
   sudo cp /opt/kopiya-iksuyemsya/kopiya-iksuyemsya.service /etc/systemd/system/
   sudo nano /etc/systemd/system/kopiya-iksuyemsya.service
   ```
   Указать: `WorkingDirectory=/opt/kopiya-iksuyemsya`, `ExecStart=/usr/bin/python3 /opt/kopiya-iksuyemsya/main.py`.

   ```bash
   sudo systemctl daemon-reload
   sudo systemctl enable kopiya-iksuyemsya.service
   sudo systemctl start kopiya-iksuyemsya.service
   ```

4. **GitHub Actions для Копия иксуюемся:** в репозитории **kopiya-iksuyemsya** в **Settings → Secrets** задать те же SSH-ключи и отдельный путь:
   - `SSH_HOST`, `SSH_USER`, `SSH_PRIVATE_KEY`, `SSH_PORT` (можно те же, что и для TradeTherapyBot)
   - `DEPLOY_PATH=/opt/kopiya-iksuyemsya`

   В этом репозитории уже есть свой `.github/workflows/deploy.yml`, который при пуше в `main` делает: `cd $DEPLOY_PATH`, `git pull`, `pip install -r requirements.txt`, `systemctl restart kopiya-iksuyemsya.service`.
