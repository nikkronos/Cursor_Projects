# Мониторинг

## Логирование
Бот ведет журнал событий в двух местах:
1. **Консоль (Terminal):** Оперативный вывод текущих действий.
2. **Файл `logs/bot.log`:** Долговременное хранение истории.

## Что отслеживать
- **Ошибки подключения к БД:** `CRITICAL` уровень в логах.
- **Ошибки API Telegram:** `ERROR` уровень (например, невозможность отправить сообщение заблокировавшему бота пользователю).
- **Действия админа:** Кто и когда изменил подписку или удалил чеки.

## Проверка работоспособности
- Если бот не отвечает на `/start` -> Проверить консоль на наличие ошибок.
- Если не работает авто-кик -> Проверить лог на наличие ошибок `apscheduler`.
