# Быстрые промпты для работы с AI агентом

## 🚀 Стартовые команды (начало работы)

### Быстрый онбординг нового агента
```
Прочитай всю базу знаний в папке docs/. После прочтения подтверди, что понял:
1. Что делает проект
2. Архитектуру
3. Правила написания кода (patterns.md)
4. Бизнес-логику
```

### Начать новую задачу/функцию
```
Есть задача: [описание задачи]

Используй шаблон из docs/specs/spec-template.md:
1. Заполни спек
2. Задай мне вопросы из секции "Какая информация нам потребуется"
3. После ответов - проведи самооценку решения (подводные камни, риски, альтернативы)
4. Создай чеклист для самопроверки
5. Разбей на шаги с тестами для каждого
```

### Проверить текущий код перед началом
```
Проведи аудит текущего кода:
- Потенциальные проблемы и уязвимости
- Неиспользуемые функции
- Несоответствия patterns.md
- Проблемы производительности
- Проблемы безопасности
```

## ⚙️ Команды во время работы

### Проверить прогресс
```
Покажи статус текущих задач из TODO списка
```

### Проверить соответствие правилам
```
Проверь текущий код на соответствие:
- patterns.md
- security.md
- Тестам должен быть написан тест для каждой функции
```

### Обновить спек после изменений
```
Обнови спек для задачи [название] с учетом изменений:
[что изменилось]
```

## ✅ Завершающие команды (конец работы)

### Завершить задачу и подготовить к коммиту
```
Задача выполнена. Проведи финальную проверку:

1. Пройдись по чеклисту из спека
2. Проверь, что все тесты написаны (предложи запустить test_manual.py)
3. Проверь соответствие patterns.md
4. Проверь логирование всех важных операций
5. Проверь, что нет hardcoded секретов
6. Обнови документацию (если нужно)
7. Проверь линтером
8. Удали временные файлы и ненужные комментарии
9. Подготовь список изменений для коммита
```

### Перед коммитом в Git
```
Перед коммитом проверь:

1. Все тесты проходят (предложи запустить test_manual.py или pytest)
2. Код соответствует patterns.md
3. Нет hardcoded секретов
4. Логирование добавлено
5. Документация обновлена (если нужно)
6. Временные файлы удалены
7. Чистый git status (нет лишних файлов)

Подготовь команды для git:
- git add [файлы]
- git commit -m "[описание]"
- git push
```

### Полная финальная проверка
```
Проведи полную финальную проверку перед коммитом:

1. ✅ Все критерии приемки из спека выполнены
2. ✅ Все тесты написаны и готовы к запуску
3. ✅ Код проверен линтером (нет ошибок)
4. ✅ Соответствие patterns.md, security.md
5. ✅ Логирование всех важных операций
6. ✅ Нет hardcoded секретов
7. ✅ Документация обновлена
8. ✅ Временные файлы удалены
9. ✅ Git готов к коммиту (список изменений)

Предложи запустить тесты если нужно.
```

## 🔄 Шаблоны для частых задач

### Добавить новую функцию
```
Новая функция: [название]

Используй workflow:
1. Создай спек в docs/specs/[название].md
2. Проведи интервью (задай вопросы)
3. Самооценка решения
4. Реализация с тестами
5. Финальная проверка
```

### Исправить баг
```
Баг: [описание]

1. Воспроизведи проблему (опиши что происходит)
2. Найди причину
3. Исправь с тестами
4. Проверь что ничего не сломалось
5. Обнови документацию если нужно
```

### Оптимизировать код
```
Оптимизация: [что оптимизируем]

1. Найди узкие места
2. Предложи решение
3. Реализуй с тестами
4. Проверь производительность
5. Обнови документацию
```

---

## 📋 Чеклист перед каждым коммитом

Используй этот чеклист в конце работы:

- [ ] Все тесты написаны
- [ ] Код соответствует patterns.md
- [ ] Нет hardcoded секретов
- [ ] Логирование добавлено
- [ ] Документация обновлена
- [ ] Линтер не находит ошибок
- [ ] Временные файлы удалены
- [ ] Git готов (проверен git status)

**Команда:** Просто напиши "чеклист перед коммитом" и агент сам пройдется по нему.

## 💡 Примеры использования

### Пример 1: Начало новой функции
```
Пользователь: Нужно добавить статистику для админа

Агент должен:
1. Прочитать docs/agent-onboarding.md (если первый раз)
2. Создать docs/specs/04-admin-statistics.md
3. Задать вопросы
4. Провести самооценку
5. Начать реализацию
```

### Пример 2: Завершение задачи
```
Пользователь: Готово, завершай

Агент должен:
1. Пройтись по чеклисту из спека
2. Проверить тесты
3. Проверить линтер
4. Обновить документацию
5. Подготовить git команды
6. Предложить запустить тесты
```

---

## 🎯 Ключевые правила для агента

1. **Всегда** используй спек-шаблон для новых задач
2. **Всегда** проводи самооценку перед началом
3. **Всегда** пиши тесты для каждой функции
4. **Всегда** проверяй чеклист перед коммитом
5. **Всегда** предлагай запустить тесты перед завершением
6. **Никогда** не коммить без проверки чеклиста
7. **Никогда** не добавляй секреты в код
