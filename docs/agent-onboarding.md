# Онбординг для AI агента

Этот документ содержит всю необходимую информацию для начала работы с проектом TradeTherapyBot.

## 📋 Краткое описание проекта

**TradeTherapyBot** - Telegram-бот для автоматизации управления доступом к закрытому сообществу трейдеров "Trade Therapy".

### Основные функции:
- Автоматический прием платежей (через проверку чеков админом)
- Контроль доступа: автоматическое добавление в группу при активной подписке и исключение (кик) при ее окончании
- Система уведомлений: напоминания за 3 дня, 1 день и 1 час до истечения подписки
- Административная панель: управление пользователями, добавление/удаление дней, просмотр чеков, статистика
- Обратная связь: сбор причин отказа от продления подписки

### Целевая аудитория:
- Трейдеры, желающие получить доступ к аналитике и сигналам сообщества
- Администратор сообщества (управление бизнесом)

## 🏗️ Архитектура проекта

Подробная информация об архитектуре: [architecture.md](../Projects/Non%20actual/TradeTherapyBot/docs/architecture.md)

### Основные компоненты:

- **`main.py`** - точка входа, инициализация БД, запуск планировщика и polling
- **`config.py`** - загрузка переменных окружения из `env_vars.txt`
- **`loader.py`** - создание объектов `bot` и `logger`, избежание циклических импортов
- **`database.py`** - вся логика работы с SQLite (подключение, таблицы, запросы, миграции)
- **`utils.py`** - утилиты для Telegram Bot API (retry, rate limiting)
- **`validators.py`** - валидация входных данных от пользователей
- **`services.py`** - бизнес-логика (планировщик подписок, исключение пользователей)
- **`handlers/`** - обработчики сообщений:
  - `helpers.py` - вспомогательные функции для создания меню и UI
  - `admin.py` - админские команды и функции
  - `user.py` - пользовательские команды и меню
  - `callbacks.py` - обработчики callback_query
  - `join_requests.py` - обработка заявок на вступление в группу

### База данных:
- **SQLite** (файл `bot.db`)
- Таблицы: `users`, `receipts`, `tariff_answers`

## 📐 Бизнес-правила

Подробная информация: [business-rules.md](../Projects/Non%20actual/TradeTherapyBot/docs/business-rules.md)

### Ключевые правила:

1. **Подписка и доступ:**
   - Доступ в группу только для пользователей со статусом `subscription_status = 'active'`
   - Срок подписки считается до конкретной даты и времени (`subscription_end_date`)

2. **Продление и оплата:**
   - Оплата подтверждается вручную администратором по чеку
   - При подтверждении оплаты подписка продлевается до 1-го числа следующего месяца
   - При ручном добавлении дней используется функция `/add_subscription_days`

3. **Уведомления и исключение:**
   - Бот проверяет подписки каждые 30 минут
   - 3 дня до конца: напоминание
   - 1 день до конца: напоминание
   - 1 час до конца: напоминание
   - Срок истек: статус → `inactive`, кик из группы, разбан для возможности вернуться

4. **Защита (Failsafe):**
   - При входе в группу бот перепроверяет подписку
   - Если подписки нет — мгновенный кик

## 💻 Паттерны и стандарты кода

Подробная информация: [docs/patterns.md](patterns.md)

### Общие правила:

- Все настройки (токены, пароли) в `env_vars.txt` через `config.py`
- SQLite через `sqlite3`, параметризованные запросы (`?`)
- Context Manager для работы с БД (`with get_db_connection() as conn:`)
- Логирование через `logging`, не использовать `print()`
- Все вызовы API Telegram и БД оборачивать в `try-except`
- Использовать `retry_telegram_api()` и `safe_send_message()` из `utils.py`

### Безопасность:
- **НИКОГДА** не хардкодить секреты в коде
- **НИКОГДА** не писать секреты в чате с агентом
- Все секреты в `env_vars.txt` (в `.gitignore`)
- Использовать только `env_vars.example.txt` для примера

### Логирование:
- Логировать все важные операции (действия админа, ошибки БД, ошибки API)
- Уровни: `DEBUG`, `INFO`, `WARNING`, `ERROR`, `CRITICAL`
- В логах не должно быть секретов

### Тестирование:
- На каждую функцию пишутся тесты
- Тесты в папке `tests/`
- Использовать `pytest`
- Тесты должны проходить перед коммитом

### Когда делать коммит:

1. **После тестирования:** Коммит делается только после того, как:
   - Все изменения протестированы
   - Убедились, что нет скрытых багов
   - Проверили связанные функции
   - Убедились, что изменения работают корректно

2. **Автоматический коммит:** После успешного тестирования агент может делать коммит автоматически (не требуется ручное подтверждение через PowerShell)

3. **Push:** После коммита автоматически выполняется `git push origin main`, что запускает CI/CD pipeline

### Пошаговая инструкция для коммита:

```powershell
# 1. Перейти в папку с репозиторием (ОБЯЗАТЕЛЬНО!)
cd "C:\Users\krono\OneDrive\Рабочий стол\Cursor_test\TradeTherapyBot"

# 2. Проверить статус (что изменилось)
git status

# 3. Добавить все изменения
git add .

# 4. Сделать коммит с описанием
git commit -m "feat: описание изменений"

# 5. Отправить в репозиторий (запустит CI/CD)
git push origin main
```

### Формат сообщений коммитов:

Используйте понятные сообщения с префиксом:
- `feat:` - новая функциональность (например: `feat: добавил тестовую кнопку "Тест" для активных участников`)
- `fix:` - исправление бага (например: `fix: исправил обработку кнопки "Тариф"`)
- `docs:` - изменения в документации (например: `docs: обновил документацию онбординга`)
- `chore:` - технические изменения (например: `chore: обновил .gitignore для исключения временных проектов`)
- `refactor:` - рефакторинг кода (например: `refactor: разделил handlers.py на модули`)
- `test:` - добавление тестов (например: `test: добавил тесты для database.py`)

**Пример многострочного коммита:**
```powershell
git commit -m "feat: добавил тестовую кнопку 'Тест' для активных участников

- Добавлена кнопка 'Тест' в главное меню для активных участников
- Добавлен обработчик кнопки, дублирующий функционал 'Тариф'
- Обновлен .gitignore для исключения ParentChildResearch/
- Создан единый документ онбординга (agent-onboarding.md)
- Обновлены правила коммитов для агента в patterns.md"
```

### CI/CD Pipeline:

**После `git push origin main` автоматически происходит:**

1. **Запуск тестов** (GitHub Actions)
   - Проверка импортов
   - Запуск pytest тестов
   - Проверка структуры handlers

2. **При успешных тестах → автоматический деплой на сервер:**
   - Подключение к серверу по SSH
   - Переход в `/opt/tradetherapybot`
   - Выполнение `git pull origin main`
   - Установка зависимостей (если изменился `requirements.txt`)
   - Перезапуск systemd service `tradetherapybot.service`
   - Проверка статуса сервиса

3. **Результат:**
   - Код обновлен на сервере
   - Бот перезапущен с новой версией
   - Все изменения применены

**Проверка результата:**
- Откройте репозиторий на GitHub: `https://github.com/nikkronos/TradeTherapyBot`
- Перейдите в раздел **Actions**
- Убедитесь, что:
  - ✅ Тесты прошли (Run Tests)
  - ✅ Деплой успешен (Deploy to Server)
