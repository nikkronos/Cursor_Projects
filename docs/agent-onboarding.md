# Онбординг нового AI агента

## Шаг 1: Чтение базы знаний
Прочитай всю документацию в папке `docs/`:
1. `project.md` - что делаем, для кого
2. `architecture.md` - структура проекта
3. `patterns.md` - правила написания кода
4. `features.md` - что уже реализовано
5. `database.md` - схема БД
6. `business-rules.md` - бизнес-логика
7. `deployment.md` - как развернуть
8. `ux-guidelines.md` - стиль интерфейса
9. `git-workflow.md` - работа с Git
10. `monitoring.md` - логи и мониторинг
11. `roadmap.md` - планы развития
12. `security.md` - правила безопасности

**Чеклист после чтения:**
- [ ] Понимаю, что делает проект
- [ ] Понимаю архитектуру
- [ ] Знаю правила написания кода
- [ ] Понимаю бизнес-логику
- [ ] Знаю правила безопасности

## Шаг 2: Анализ текущего кода
Изучи основные файлы:
- `main.py` - точка входа
- `handlers.py` - обработчики сообщений
- `services.py` - бизнес-логика
- `database.py` - работа с БД
- `config.py` - конфигурация
- `loader.py` - инициализация

**Чеклист:**
- [ ] Понимаю структуру кода
- [ ] Понимаю, как работают ключевые функции
- [ ] Нашел потенциальные проблемы (зафиксируй их)

## Шаг 3: Получение задачи
Получи от пользователя задачу:
- Что нужно сделать?
- Какие есть требования?
- Какие ограничения?

## Шаг 4: Создание спека
1. Открой шаблон `docs/specs/spec-template.md`
2. Заполни все секции
3. Задай пользователю вопросы из секции "Какая информация нам потребуется"
4. После ответов - заверши спецификацию

**Чеклист:**
- [ ] Спецификация заполнена
- [ ] Все вопросы заданы и получены ответы
- [ ] Описаны тесты для каждого шага
- [ ] Описан план безопасности

## Шаг 5: Оценка плана
**ВАЖНО:** Перед началом работы проведи самооценку:

### Оценка решения
- Это действительно лучшее решение? Почему?
- Какие есть подводные камни?
- Какие проблемы могут возникнуть?
- Какие есть риски?
- Как еще можно решить эту задачу?

### Критерии оценки и чеклист
Для каждой задачи из спека:
- Критерий успеха: ...
- Чеклист самопроверки:
  - [ ] ...
  - [ ] ...

## Шаг 6: Реализация
1. Создай ветку (если работаешь с Git)
2. Реализуй по шагам из спека
3. После каждого шага:
   - Напиши тесты
   - Запусти тесты
   - Проверь логирование
   - Проверь соответствие patterns.md
4. Исправь найденные проблемы

## Шаг 7: Самопроверка
Пройдись по чеклисту из спека:
- [ ] Все критерии приемки выполнены
- [ ] Все тесты проходят
- [ ] Код соответствует patterns.md
- [ ] Добавлено логирование
- [ ] Нет hardcoded секретов
- [ ] Документация обновлена

## Шаг 8: Аудит кода
Попроси себя провести аудит:
- Потенциальные проблемы и уязвимости
- Неиспользуемые функции
- Несоответствия patterns.md
- Проблемы производительности
- Проблемы безопасности

## Шаг 9: Рефакторинг (если нужно)
Если аудит выявил проблемы - исправь их и обнови спеки.

## Шаг 10: Коммит
- Убедись, что все тесты проходят
- Закоммить изменения с понятным сообщением
- GitHub Actions автоматически запустит тесты (когда будет настроено)
