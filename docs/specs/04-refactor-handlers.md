# Техническое задание: Разделение handlers.py на модули

## Что надо сделать
Разделить монолитный файл `handlers.py` (1356 строк) на модульную структуру для улучшения поддерживаемости и читаемости кода. Создать пакет `handlers/` с отдельными модулями по функциональности.

## Как это должно работать
После рефакторинга структура будет следующей:
- `handlers/__init__.py` - экспортирует все обработчики для регистрации
- `handlers/admin.py` - админские команды и функции (backup, управление пользователями, чеки, оплаты)
- `handlers/user.py` - пользовательские команды и меню (start, тарифы, правила, статус, обратная связь)
- `handlers/callbacks.py` - обработчики callback_query (подтверждение/отклонение оплат, тарифов, причин)
- `handlers/join_requests.py` - обработка заявок на вступление в группу и chat_member обновления
- `handlers/helpers.py` - вспомогательные функции (send_main_menu, send_admin_menu, send_payment_info и т.д.)

Все обработчики должны регистрироваться автоматически при импорте пакета `handlers` в `main.py`.

## Потенциальные проблемы и риски
- **Риск 1:** Циклические импорты между модулями
  - **Митигация:** Использовать импорты только в нужных местах, helpers не должен импортировать другие модули handlers
- **Риск 2:** Декораторы не регистрируются при импорте
  - **Митигация:** Убедиться, что все модули импортируются в __init__.py, а __init__.py импортируется в main.py
- **Риск 3:** Потеря функций при переносе
  - **Митигация:** Внимательно проверить каждый шаг, использовать grep для поиска всех функций
- **Риск 4:** Ошибки в импортах (неправильные пути)
  - **Митигация:** Использовать относительные импорты внутри пакета, абсолютные для внешних модулей
