# Техническое задание: Оптимизация базы данных (индексы)

## Что надо сделать
Добавить индексы в базу данных SQLite для ускорения запросов по статусу подписки и датам окончания. Это критично для производительности, так как планировщик проверяет подписки каждые 30 минут и делает запросы по этим полям.

## Как это должно работать
При инициализации БД будут созданы индексы на поля:
- `subscription_status` - для быстрого поиска активных/неактивных пользователей
- `subscription_end_date` - для быстрого поиска подписок по дате окончания
- Комбинированный индекс на `(subscription_status, subscription_end_date)` - для запросов планировщика

Для существующих баз данных будет создана функция миграции, которая добавит индексы без потери данных.

## Какие инструменты нам понадобятся
- SQLite (уже используется)
- Стандартная библиотека `sqlite3` (уже используется)
- Никаких новых зависимостей не требуется

## Потенциальные проблемы и риски
- **Риск 1:** Миграция может занять время на большой БД
  - **Митигация:** Логировать прогресс, использовать `IF NOT EXISTS`
- **Риск 2:** SQLite может иметь ограничения на количество индексов
  - **Митигация:** Проверить документацию SQLite (обычно лимит высокий)
- **Риск 3:** Старые версии SQLite могут не поддерживать некоторые функции
  - **Митигация:** Использовать стандартный синтаксис SQLite
