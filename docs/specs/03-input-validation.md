# Техническое задание: Валидация входных данных

## Что надо сделать
Добавить валидацию всех входных данных от пользователей перед использованием. Это критично для безопасности и стабильности бота - некорректные данные могут привести к ошибкам, SQL-инъекциям (хотя используются параметризованные запросы) или неожиданному поведению.

## Как это должно работать

### Валидация пользовательских ID
- Проверка, что ID является положительным целым числом
- Проверка диапазона (например, Telegram ID обычно в диапазоне 1-2^63)
- Обработка некорректных значений с понятными сообщениями пользователю

### Валидация дней подписки
- Проверка, что значение является целым числом
- Проверка диапазона (например, от -365 до 365 дней)
- Обработка отрицательных значений (если допустимо)

### Валидация текстовых полей
- Проверка длины строки (максимальная длина для безопасности)
- Проверка на запрещенные символы (если нужно)
- Санитизация (удаление лишних пробелов, переносов строк)

### Валидация дат
- Проверка формата даты
- Проверка, что дата в разумном диапазоне (не в прошлом для окончания подписки, не слишком далеко в будущем)

## Какие инструменты нам понадобятся
- Стандартная библиотека Python (`re` для regex, если нужно)
- Никаких новых зависимостей не требуется

## Тесты

### Тест 1: Валидация user_id
**Что тестируем:** Функция `validate_user_id()`
**Ожидаемый результат:** 
- Принимает корректные ID (положительные числа)
- Отклоняет отрицательные числа
- Отклоняет нечисловые значения
- Отклоняет слишком большие числа

### Тест 2: Валидация дней
**Что тестируем:** Функция `validate_days()`
**Ожидаемый результат:**

### Тест 3: Валидация текста
**Что тестируем:** Функция `validate_text()`
**Ожидаемый результат:**

## Потенциальные проблемы и риски
- **Риск 1:** Слишком строгая валидация может блокировать легитимные вводы
  - **Митигация:** Протестировать на реальных данных, сделать валидацию гибкой
- **Риск 2:** Валидация может усложнить код
  - **Митигация:** Вынести в отдельный модуль, использовать функции-хелперы

## Альтернативные решения

- Вариант Б: Использовать готовую библиотеку валидации (минусы: новая зависимость)
