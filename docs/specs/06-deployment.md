# Техническое задание: Проверка и развертывание бота на сервере

## Что надо сделать
Проверить текущее состояние развертывания бота на сервере и, при необходимости, настроить автоматический запуск через systemd для работы 24/7 без необходимости включенного ПК.

## Как это должно работать
После развертывания:
- Бот должен работать на сервере 24/7
- Бот должен автоматически запускаться при перезагрузке сервера
- Бот должен автоматически перезапускаться при падении
- Логи должны быть доступны через systemd journal
- Бот должен быть доступен через команду `/start` в Telegram

## Какие инструменты нам понадобятся
- Доступ к серверу (SSH)
- systemd (стандартный в Linux)
- Python 3.7+ на сервере
- Файл `tradetherapybot.service` (уже есть в проекте)
- Файл `env_vars.txt` с настройками на сервере

### Шаг 3: Настройка systemd service
**Что сделать:** 
- Скопировать `tradetherapybot.service` в `/etc/systemd/system/`
- Обновить пути в файле service (WorkingDirectory, ExecStart, User)
- Перезагрузить конфигурацию systemd: `systemctl daemon-reload`
- Включить автозапуск: `systemctl enable tradetherapybot.service`

**Что проверить:** 
- Файл service создан в правильном месте
- Пути в файле корректны
- Service включен для автозапуска

**Тесты:** 
- Проверить статус: `systemctl status tradetherapybot.service`
- Проверить, что service включен: `systemctl is-enabled tradetherapybot.service`

**Логирование:** 
- Записать команды и результаты в документацию

- **Риск 1:** Бот не запускается из-за отсутствия зависимостей

- **Риск 2:** Неправильные пути в service файле
  - **Митигация:** Тщательно проверить пути перед запуском
- **Риск 3:** Проблемы с правами доступа
  - **Митигация:** Создать отдельного пользователя для бота, настроить права
- **Риск 4:** Бот падает из-за ошибок в коде
  - **Митигация:** Проверить логи, исправить ошибки, service автоматически перезапустит
