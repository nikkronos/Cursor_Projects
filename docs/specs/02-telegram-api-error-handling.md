# Техническое задание: Улучшение обработки ошибок Telegram API

## Что надо сделать
Добавить надежную обработку ошибок при вызовах Telegram Bot API: автоматические повторы (retry) при временных ошибках и корректная обработка rate limiting (429 ошибка). Это критично для стабильности бота, так как Telegram API может временно быть недоступен или ограничивать количество запросов.

## Как это должно работать

### Retry механизм
- При ошибках типа `TelegramApiException`, `ConnectionError`, `Timeout` - автоматически повторять запрос до 3 раз с экспоненциальной задержкой (1 сек, 2 сек, 4 сек)
- Логировать каждую попытку повторения
- После исчерпания попыток - выбрасывать исключение

### Rate Limiting (429 ошибка)
- При получении ошибки 429 (Too Many Requests) - извлекать `retry_after` из ответа
- Ждать указанное время перед повтором
- Логировать rate limit события

### Применение к критичным операциям
- `bot.send_message()` - отправка сообщений пользователям
- `bot.approve_chat_join_request()` - одобрение заявок
- `bot.kick_chat_member()` - удаление пользователей
- `bot.get_chat_member()` - проверка статуса пользователя

## Потенциальные проблемы и риски
- **Риск 1:** Retry может увеличить время ответа бота
  - **Митигация:** Ограничить количество попыток и время задержек
- **Риск 2:** Некорректная обработка rate limiting может привести к банам
  - **Митигация:** Строго соблюдать retry_after из ответа API
- **Риск 3:** Слишком много повторений может перегрузить API
  - **Митигация:** Использовать экспоненциальную задержку и лимит попыток
