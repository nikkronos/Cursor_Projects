# Техническое задание: Добавление type hints

## Что надо сделать
Добавить type hints (аннотации типов) во все функции всех модулей проекта для улучшения читаемости кода, поддержки IDE и возможности статической проверки типов с помощью mypy.

## Как это должно работать
После добавления type hints:
- Все функции будут иметь аннотации типов для параметров и возвращаемых значений
- Для сложных типов (списки, словари, опциональные значения) будут использоваться типы из `typing`
- Код останется полностью обратно совместимым (type hints не влияют на выполнение)
- IDE сможет лучше подсказывать типы и находить ошибки
- В будущем можно будет использовать mypy для статической проверки типов

## Какие инструменты нам понадобятся
- Стандартная библиотека Python: `typing` (уже используется в `utils.py`)
- Опционально: `mypy` для проверки типов (можно установить позже)
- Типы из `telebot` (если доступны) или `typing.Any` для объектов бота

## Потенциальные проблемы и риски
- **Риск 1:** Неправильные типы могут ввести в заблуждение
  - **Митигация:** Тщательно проверять типы, использовать `typing.Any` где тип неизвестен
- **Риск 2:** Циклические импорты при использовании типов
  - **Митигация:** Использовать строковые аннотации (`"Type"`) или `TYPE_CHECKING` из typing
- **Риск 3:** Несовместимость с версией Python
  - **Митигация:** Использовать стандартные типы из `typing`, доступные в Python 3.7+

## Альтернативные решения
- Вариант А: Использовать только базовые типы (int, str, bool)
  - **Минусы:** Меньше пользы, не покрывает сложные типы
- Вариант Б: Использовать полную типизацию с Protocol и Generic
  - **Минусы:** Сложнее, может быть избыточно для текущего проекта
- **Выбран:** Полная типизация с использованием стандартных типов из `typing`, где необходимо - `Any` или строковые аннотации
