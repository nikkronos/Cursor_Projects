# Техническое задание: Rate limiting для защиты от спама

## Что надо сделать
Реализовать механизм rate limiting (ограничение частоты запросов) для защиты бота от злоупотреблений и спама. Ограничить количество запросов от одного пользователя в единицу времени и временно блокировать пользователей, превышающих лимит.

## Как это должно работать
- Пользователь отправляет сообщение/команду → система проверяет количество его запросов за последние N секунд
- Если запросов меньше лимита → запрос обрабатывается нормально
- Если запросов больше лимита → пользователю отправляется предупреждение, запрос игнорируется
- При многократном превышении лимита → временная блокировка пользователя (например, на 5 минут)
- Администратор не ограничивается rate limiting
- Логирование всех случаев превышения лимита

**Примеры:**
- Пользователь отправляет 10 сообщений за 10 секунд → после 5-го сообщения получает предупреждение, остальные игнорируются
- Пользователь продолжает спамить → получает временную блокировку на 5 минут
- Администратор может отправлять любое количество сообщений без ограничений

## Какие инструменты нам понадобятся
- Стандартная библиотека Python: `time`, `collections.deque` или `dict` для хранения истории запросов
- Модуль `utils.py` для добавления функции rate limiting
- Опционально: можно использовать `collections.defaultdict` для удобства

## Потенциальные проблемы и риски
- **Риск 1:** Лимиты слишком строгие - обычные пользователи не могут нормально использовать бота
  - **Митигация:** Начать с мягких лимитов (например, 10 запросов за 10 секунд), можно настроить позже
- **Риск 2:** Хранение истории запросов в памяти может привести к утечке памяти
  - **Митигация:** Регулярно очищать старые записи, использовать ограничение по времени
- **Риск 3:** Rate limiting может замедлить обработку запросов
  - **Митигация:** Использовать эффективные структуры данных (dict, deque), минимизировать проверки
- **Риск 4:** При перезапуске бота история запросов теряется
  - **Митигация:** Это нормально, блокировки сбросятся при перезапуске (можно добавить сохранение в БД позже, если нужно)
