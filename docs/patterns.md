# Паттерны и Стандарты кода

## Общие правила
- Все настройки (токены, пароли) выносятся в `env_vars.txt` и загружаются через `config.py`.
- Прямые SQL-запросы используются через библиотеку `psycopg2`. Важно использовать параметризованные запросы (`%s`), чтобы избежать SQL-инъекций.
- Логирование через модуль `logging`. `print()` не используется для продакшн-логов.

## Обработка ошибок
- Все вызовы API Telegram и БД оборачиваются в блоки `try-except`.
- Ошибки логируются с уровнем `ERROR` или `CRITICAL`.
- При ошибке отправки сообщения пользователю, работа бота не должна прерываться.

## Структура хендлеров
- Используются декораторы `@bot.message_handler`.
- Хендлеры разделены по смыслу (админские, пользовательские, колбэки).
- Для многошаговых диалогов используется `bot.register_next_step_handler`.
