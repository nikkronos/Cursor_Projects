# 🛡️ Предотвращение потери файлов

## ⚠️ Проблема

Файлы могут быть случайно очищены или удалены при:
- Случайном удалении содержимого при редактировании
- Неправильной операции "Select All" + Delete
- Массовой замене через "Find & Replace"
- Проблемах в IDE/редакторе
- Неправильном разрешении конфликтов Git

**Важно:** Всегда можно восстановить файлы из истории Git, если они были закоммичены!

---

## ✅ Правила работы с Git

### 1. Регулярные коммиты (самое важное!)

**Правило:** Коммитьте после каждой законченной задачи!

```bash
git add .
git commit -m "описание изменений"
git push origin main
```

**Преимущества:**
- Полная история всех изменений
- Можно восстановить любую версию
- GitHub служит резервной копией
- Если файл потерян - можно восстановить

### 2. Регулярный push в GitHub

**Отправляйте изменения сразу после коммита:**
- Это создает резервную копию в облаке
- Можно восстановить даже если локальные файлы потеряны
- История сохраняется на сервере

### 3. Проверка перед коммитом

**Всегда проверяйте статус:**
```bash
git status        # Что изменилось
git diff          # Какие изменения
```

**Проверка пустых файлов (PowerShell):**
```powershell
Get-ChildItem -Recurse -File *.py | ForEach-Object {
    $lines = (Get-Content $_.FullName).Count
    if ($lines -lt 5) {
        Write-Host "⚠️  ВНИМАНИЕ: $($_.FullName) содержит только $lines строк!"
    }
}
```

- [ ] Нет случайных изменений

## 🔧 Восстановление файла из истории

Если файл случайно очистился или удалился:

### Шаг 1: Найти коммит, где файл был

```bash
git log --all --full-history -- <путь_к_файлу>
```

Пример:
```bash
git log --all --full-history -- database.py
```

### Шаг 2: Восстановить файл

```bash
git checkout <commit-hash> -- <путь_к_файлу>
```

Пример:
```bash
git checkout fe5a9f93bda877a15a479b4db401b9b94cc3307a -- database.py
```

### Шаг 3: Проверить восстановление

```powershell
(Get-Content database.py).Count  # Должно быть > 0
Get-Content database.py -TotalCount 10  # Посмотреть первые строки
```

### Шаг 4: Закоммитить восстановление

```bash
git add database.py
git commit -m "fix: восстановление database.py из истории Git"
git push origin main
```

## 🚨 Что делать, если файл потерян

1. **Проверить историю Git:**
   ```bash
   git log --all --full-history -- <путь_к_файлу>
   ```

2. **Восстановить из истории:**
   ```bash
   git checkout <commit-hash> -- <путь_к_файлу>
   ```

3. **Если файла нет в Git - взять с сервера:**
   ```bash
   ssh root@81.200.146.32
   cat /opt/tradetherapybot/<файл>
   ```

4. **Проверить GitHub:**
   - Файл может быть в последней версии на GitHub
   - Можно скачать через веб-интерфейс

## 💡 Рекомендации

1. **Коммитьте часто** - после каждого значимого изменения
2. **Push регулярно** - хотя бы раз в день
3. **Используйте ветки** - для экспериментов создавайте отдельные ветки
4. **Проверяйте перед коммитом** - `git status` и `git diff`
5. **Будьте осторожны** - с массовыми операциями (Replace All, массовое удаление)

## 🔍 Как проверить текущее состояние

### Проверить статус Git:
```bash
git status
```

### Проверить, нет ли пустых файлов:
```powershell
Get-ChildItem -Recurse -File | Where-Object { 
    (Get-Content $_.FullName -Raw).Length -lt 10 
} | Select-Object FullName
```

### Проверить важные файлы проекта:
- `database.py` - должен быть > 200 строк
- `services.py` - должен быть > 100 строк
- `handlers/*.py` - должны содержать код
- `main.py` - должен содержать код
- `config.py` - должен содержать код
