# Стратегия коммитов

## Когда делать коммиты

### Вариант 1: По окончанию всех задач (рекомендуется)

**Плюсы:**
- Один большой коммит с логически связанными изменениями
- Проще отслеживать изменения
- Меньше коммитов в истории

**Минусы:**
- Если что-то сломается, сложнее найти проблему
- Больше изменений в одном коммите

**Когда использовать:**
- Когда работаете над одной большой задачей
- Когда все изменения логически связаны
- Когда уверены, что всё работает

### Вариант 2: После каждой завершённой подзадачи

**Плюсы:**
- Легче откатить конкретное изменение
- Понятнее история изменений
- Безопаснее (меньше риск потерять работу)

**Минусы:**
- Много мелких коммитов
- Может быть сложнее понять общую картину

**Когда использовать:**
- Когда работаете над несколькими независимыми задачами
- Когда хотите сохранить промежуточные результаты
- Когда есть риск сломать что-то важное

```powershell

```

### 2. Добавить файлы


```powershell

```

### 4. Отправить в GitHub

```powershell
git push origin main
```
