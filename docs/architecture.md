# Архитектура и Структура Проекта

Проект разделен на модули для удобства поддержки и расширения.

## Структура файлов

*   **`main.py`**: Точка входа. Инициализирует базу данных, запускает планировщик задач и начинает опрос серверов Telegram (polling).
*   **`config.py`**: Отвечает за загрузку переменных окружения (токен, ID админа) из файла `.env` или `env_vars.txt`.
*   **`loader.py`**: Создает основные объекты: `bot` (экземпляр TeleBot) и настроенный `logger`. Используется для избежания циклических импортов.
*   **`database.py`**: Содержит всю логику работы с SQLite: подключение, создание таблиц, функции `SELECT`/`INSERT`/`UPDATE`, миграции.
*   **`utils.py`**: Утилиты для работы с Telegram Bot API: механизм retry для обработки временных ошибок и rate limiting.
*   **`validators.py`**: Модуль валидации входных данных от пользователей: проверка ID, дней подписки, текстовых полей.
*   **`services.py`**: Бизнес-логика, не связанная напрямую с обработкой сообщений:
    *   Планировщик (`scheduler`) для проверки подписок.
    *   Логика исключения пользователей из чата.
    *   Логика добавления/удаления дней подписки.
*   **`handlers.py`**: Содержит все функции-обработчики сообщений (`@bot.message_handler`). Отвечает за взаимодействие с пользователем (меню, кнопки, админка).

## База Данных

Используется **SQLite**. Файл `bot.db` создается в корневой директории.

### Таблицы:
1.  `users`: Хранит информацию о пользователях, статусе подписки и сроках.
2.  `receipts`: История присланных чеков (file_id).

## Резервное копирование
Реализована команда `/backup` (только для админа), которая отправляет актуальный файл `bot.db` в чат.
