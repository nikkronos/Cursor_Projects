## 📍 ШАГ 3: Создать SSH ключ

Скопируйте и вставьте эту команду в PowerShell (нажмите Enter):

```powershell
powershell -ExecutionPolicy Bypass -File ".\scripts\setup-ssh-key.ps1"
```

**Что должно произойти:** 
- Скрипт создаст SSH ключ
- Покажет вам два ключа:
  - **ПРИВАТНЫЙ** ключ (красный текст) - сохраните его, он нужен для GitHub
  - **ПУБЛИЧНЫЙ** ключ (зеленый текст) - сохраните его, он нужен для сервера

**⚠️ ВАЖНО:** Скопируйте оба ключа куда-нибудь (в блокнот), они понадобятся дальше!

## 📍 ШАГ 4: Добавить публичный ключ на сервер

### 4.1. Подключиться к серверу

В **НОВОМ** окне PowerShell (не закрывайте предыдущее!) выполните:

```powershell
ssh root@81.200.146.32
```

Вас попросят ввести пароль - введите пароль от сервера.

### 4.2. На сервере выполнить команды

После подключения к серверу, выполните эти команды по очереди (каждую отдельно):

```bash
mkdir -p ~/.ssh
```

```bash
nano ~/.ssh/authorized_keys
```

### 4.3. Вставить публичный ключ (простой способ)

**Способ 1 (рекомендуется):** Используйте команду `echo` - это проще!

1. Скопируйте ваш публичный ключ из PowerShell (где вы запускали скрипт)

2. Выполните команду на сервере, **заменив `ВАШ_ПУБЛИЧНЫЙ_КЛЮЧ`** на ваш реальный ключ:
   ```bash
   echo 'ВАШ_ПУБЛИЧНЫЙ_КЛЮЧ' >> ~/.ssh/authorized_keys
   ```
   
   **Пример:** Если ваш ключ начинается с `ssh-ed25519 AAAAC3...`, то команда будет:
   ```bash
   echo 'ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIMLMrYP43Kf4Cbk2WDRJ1xkBktKxB/bx95Ydj1dI4x8o github-actions-deploy' >> ~/.ssh/authorized_keys
   ```

**Способ 2 (альтернатива):** Использовать редактор nano

1. Выполните команду:
   ```bash
   nano ~/.ssh/authorized_keys
   ```

2. В открывшемся редакторе:
   - Прокрутите мышкой до самого конца файла (если там что-то есть)
   - Или просто нажмите **Enter** несколько раз
   - **Вставьте** публичный ключ (Ctrl + V или правая кнопка мыши)
   - Нажмите **Enter** еще раз
   - Сохранить: **Ctrl + O**, затем **Enter**
   - Выйти: **Ctrl + X**

### 4.4. Установить права доступа

Выполните эту команду:

```bash
chmod 600 ~/.ssh/authorized_keys
```

### 4.5. Выйти с сервера

Выполните:

```bash
exit
```

## 📍 ШАГ 5: Настроить GitHub Secrets

### 5.1. Открыть GitHub

1. Откройте браузер
2. Перейдите на https://github.com
3. Войдите в свой аккаунт
4. Откройте ваш репозиторий (TradeTherapyBot)

### 5.2. Перейти в настройки секретов

1. В репозитории нажмите на вкладку **Settings** (вверху справа)
2. В левом меню найдите **Secrets and variables**
3. Нажмите **Actions**
4. Нажмите кнопку **New repository secret**

### 5.3. Добавить секреты

Добавьте каждый секрет отдельно (нажимайте **New repository secret** для каждого):

#### Секрет 1: SSH_PRIVATE_KEY
- **Name:** `SSH_PRIVATE_KEY`
- **Value:** Вставьте **ПРИВАТНЫЙ ключ** из шага 3 (весь текст от `-----BEGIN` до `-----END`)
- Нажмите **Add secret**

#### Секрет 2: SSH_HOST
- **Name:** `SSH_HOST`
- **Value:** `81.200.146.32`
- Нажмите **Add secret**

#### Секрет 3: SSH_PORT
- **Name:** `SSH_PORT`
- **Value:** `22`
- Нажмите **Add secret**

#### Секрет 4: SSH_USER
- **Name:** `SSH_USER`
- **Value:** `root`
- Нажмите **Add secret**

#### Секрет 5: DEPLOY_PATH
- **Name:** `DEPLOY_PATH`
- **Value:** `/opt/tradetherapybot`
- Нажмите **Add secret**

#### Секрет 6: SERVICE_NAME
- **Name:** `SERVICE_NAME`
- **Value:** `tradetherapybot`
- Нажмите **Add secret**

## 📍 ШАГ 6: Проверить работу (тестовый деплой)

### 6.1. Вернуться в PowerShell в папке проекта

Если PowerShell был закрыт, откройте его заново и выполните:

```powershell
cd "C:\Users\krono\OneDrive\Рабочий стол\Cursor_test\TradeTherapyBot"
```

### 6.2. Сделать тестовый коммит

Выполните эти команды по очереди:

```powershell
git add .
```

```powershell
git commit -m "ci: настройка автоматического деплоя"
```

```powershell
git push origin main
```

**Что должно произойти:**
- Код отправится в GitHub
- GitHub автоматически запустит тесты
- Если тесты пройдут, автоматически запустится деплой

### 6.3. Проверить результат

1. Откройте ваш репозиторий на GitHub
2. Нажмите на вкладку **Actions** (вверху)
3. Вы увидите два процесса:
   - **Run Tests** - должен быть зеленым ✅
   - **Deploy to Server** - должен быть зеленым ✅

Если оба зеленые - всё работает! 🎉

## ❓ Если что-то пошло не так

### Проблема: "Файл не найден"
**Решение:** Убедитесь, что вы в правильной папке. Выполните:
```powershell
cd "C:\Users\krono\OneDrive\Рабочий стол\Cursor_test\TradeTherapyBot"
```

### Проблема: "Ошибка SSH подключения"
**Решение:** 
1. Проверьте, что публичный ключ правильно добавлен на сервер
2. Попробуйте подключиться вручную:
```powershell
ssh root@81.200.146.32
```

### Проблема: "Деплой не запустился"
**Решение:**
1. Проверьте, что все секреты добавлены в GitHub
2. Проверьте логи в разделе Actions на GitHub
