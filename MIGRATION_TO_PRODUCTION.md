## Шаг 1: Получение токена бота

1. Откройте Telegram и найдите бота **@BotFather**
2. Отправьте команду `/mybots`
3. Выберите бота **@tradetherapybot**
4. Выберите **API Token**
5. Скопируйте токен (формат: `1234567890:ABCdefGHIjklMNOpqrsTUVwxyz`)

⚠️ **ВАЖНО:** Токен — это секретная информация! Не публикуйте его в Git!

## Шаг 2: Получение ID администратора

Ваш Telegram ID нужен для доступа к админ-панели бота.

### Способ 1: Через бота
1. Запустите бота (даже тестового)
2. Отправьте команду `/me`
3. Бот ответит ваш ID

### Способ 2: Через @userinfobot
1. Найдите бота **@userinfobot** в Telegram
2. Отправьте ему любое сообщение
3. Он покажет ваш ID

## Шаг 3: Получение ID группы (когда группа будет готова)

1. Добавьте бота **@userinfobot** в вашу группу
2. Отправьте в группу любое сообщение
3. Бот покажет ID группы (формат: `-1001234567890`)

**Или:**

1. Добавьте бота **@RawDataBot** в группу
2. Отправьте `/start`
3. Найдите `"chat":{"id":-1001234567890}` — это ID группы

## Шаг 4: Настройка на сервере

### Вариант А: Если бот уже запущен на сервере (замена тестового)

1. **Подключитесь к серверу:**
   ```bash
   ssh root@81.200.146.32
   ```

2. **Перейдите в папку бота:**
   ```bash
   cd /opt/tradetherapybot
   ```

3. **Отредактируйте файл `env_vars.txt`:**
   ```bash
   nano env_vars.txt
   ```

4. **Замените значения:**
   ```
   BOT_TOKEN=ваш_токен_от_BotFather
   ADMIN_ID=ваш_telegram_id
   GROUP_CHAT_ID=id_группы_когда_будет_готова
   ```

5. **Сохраните файл:** `Ctrl+O`, `Enter`, `Ctrl+X`

6. **Перезапустите бота:**
   ```bash
   systemctl restart tradetherapybot.service
   ```

7. **Проверьте статус:**
   ```bash
   systemctl status tradetherapybot.service
   ```

8. **Проверьте логи:**
   ```bash
   journalctl -u tradetherapybot.service -n 50 --no-pager
   ```

## ⚠️ Важные ограничения без прав администратора в группе

Если бот **НЕ является администратором** группы, следующие функции **НЕ будут работать**:

### ❌ Не работает:
1. **Автоматическое удаление пользователей** (`kick_chat_member`)
   - Бот не сможет автоматически удалять пользователей с истекшей подпиской
   - **Решение:** Удалять вручную или сделать бота админом

2. **Автоматическое одобрение/отклонение заявок** (`approve_chat_join_request` / `decline_chat_join_request`)
   - Бот не сможет автоматически одобрять заявки на вступление
   - **Решение:** Одобрять вручную или сделать бота админом

3. **Failsafe kick** (защита от входа по ссылке)
   - Бот не сможет удалить пользователя, который вступил по ссылке без подписки
   - **Решение:** Сделать бота админом или контролировать вручную

4. **Получение списка администраторов** (`get_chat_administrators`)
   - Функция может не работать
   - **Решение:** Не критично, используется только для отображения

### ✅ Работает:
1. **Все пользовательские функции:**
   - `/start`, меню, тарифы, правила
   - Отправка чеков
   - Просмотр статуса подписки
   - Обратная связь

2. **Админ-панель:**
   - Просмотр пользователей
   - Добавление/удаление дней подписки
   - Подтверждение/отклонение чеков
   - Просмотр истории чеков
   - Бэкап базы данных

3. **Уведомления:**
   - Напоминания об истечении подписки
   - Уведомления о подтверждении оплаты

4. **Проверка статуса пользователя:**
   - `get_chat_member` — проверка, является ли пользователь участником группы

## Рекомендации

### Для полной функциональности:

1. **Сделайте бота администратором группы:**
   - Откройте настройки группы
   - Перейдите в "Администраторы"
   - Добавьте @tradetherapybot
   - Включите права:
     - ✅ Удалять участников
     - ✅ Одобрять заявки на вступление
     - ✅ Приглашать пользователей

2. **Настройте группу:**
   - Включите "Одобрение заявок" (если нужно)
   - Убедитесь, что бот может видеть заявки

### Если бот не может быть админом:

- Все основные функции будут работать
- Автоматическое удаление пользователей нужно будет делать вручную
- Заявки на вступление нужно будет одобрять вручную
- Бот будет логировать попытки удаления, но они будут завершаться ошибкой

## Миграция данных (если нужно)

Если у вас есть данные в тестовом боте и нужно их перенести:

1. **Сделайте бэкап тестовой БД:**
   ```bash
   # На сервере с тестовым ботом
   cp /opt/tradetherapybot/bot.db /opt/tradetherapybot/bot.db.backup
   ```

2. **Скопируйте БД на новый сервер/в новую папку:**
   ```bash
   scp /opt/tradetherapybot/bot.db root@81.200.146.32:/opt/tradetherapybot/bot.db
   ```

3. **Убедитесь, что права доступа правильные:**
   ```bash
   chown root:root /opt/tradetherapybot/bot.db
   chmod 644 /opt/tradetherapybot/bot.db
   ```
