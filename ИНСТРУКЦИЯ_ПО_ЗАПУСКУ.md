# Пошаговая инструкция по запуску бота на сервере

## Шаг 1: Подготовка файлов на вашем компьютере

У вас уже есть файл `tradetherapybot.service` в папке проекта `TradeTherapyBot`.

Этот файл нужно будет:
1. Скопировать на сервер
2. Отредактить на сервере
3. Поместить в системную папку

## Шаг 2: Подключение к серверу

### 2.1. Если у вас Linux VPS/сервер:
Откройте терминал на вашем компьютере (или используйте PuTTY для Windows) и подключитесь к серверу:

```bash
ssh ваш_пользователь@ip_адрес_сервера
```

Например:
```bash
ssh root@192.168.1.100
```

### 2.2. Если у вас Windows-сервер:
Используйте удаленный рабочий стол (RDP) для подключения.

## Шаг 3: Загрузка бота на сервер

### Вариант А: Через SCP (если у вас Linux-сервер)

Откройте новый терминал на вашем компьютере (не на сервере!) и выполните:

```bash
# Перейдите в папку с ботом на вашем компьютере
cd "C:\Users\krono\OneDrive\Рабочий стол\Cursor_test\TradeTherapyBot"

# Скопируйте всю папку на сервер
scp -r * ваш_пользователь@ip_адрес_сервера:/opt/tradetherapybot/
```

Например:
```bash
scp -r * root@192.168.1.100:/opt/tradetherapybot/
```

### Вариант Б: Через SFTP клиент (FileZilla, WinSCP)

1. Установите FileZilla или WinSCP
2. Подключитесь к серверу
3. Скопируйте все файлы из папки `TradeTherapyBot` в `/opt/tradetherapybot/` на сервере

### Вариант В: Через git (если есть репозиторий)

```bash
ssh ваш_пользователь@ip_адрес_сервера
cd /opt
git clone ваш_репозиторий tradetherapybot
```

## Шаг 4: Настройка на сервере

### 4.1. Подключитесь к серверу по SSH

```bash
ssh ваш_пользователь@ip_адрес_сервера
```

### 4.2. Создайте папку для бота (если еще не создали)

```bash
sudo mkdir -p /opt/tradetherapybot
cd /opt/tradetherapybot
```

### 4.3. Проверьте, что файлы скопировались

```bash
ls -la
```

Должны быть видны файлы: `main.py`, `handlers.py`, `tradetherapybot.service` и другие.

### 4.4. Установите зависимости Python

```bash
cd /opt/tradetherapybot
python3 --version  # Проверьте, что Python 3 установлен

# Установите зависимости
pip3 install -r requirements.txt
```

Если команда `pip3` не найдена, попробуйте:
```bash
python3 -m pip install -r requirements.txt
```

### 4.5. Настройте файл с переменными окружения

```bash
nano env_vars.txt
```

Или используйте другой редактор:
```bash
vi env_vars.txt
```

Убедитесь, что в файле указаны:
```
BOT_TOKEN=ваш_токен_бота
ADMIN_ID=ваш_id_телеграм
GROUP_CHAT_ID=id_закрытой_группы
```

Сохраните файл:
- В `nano`: нажмите `Ctrl+O` (сохранить), затем `Enter`, затем `Ctrl+X` (выход)
- В `vi`: нажмите `Esc`, затем введите `:wq` и `Enter`

## Шаг 5: Настройка systemd service

### 5.1. Скопируйте service файл в системную папку

```bash
sudo cp /opt/tradetherapybot/tradetherapybot.service /etc/systemd/system/
```

### 5.2. Откройте файл для редактирования

```bash
sudo nano /etc/systemd/system/tradetherapybot.service
```

### 5.3. Измените пути в файле

Найдите эти строки и измените пути:

**Было:**
```
WorkingDirectory=/path/to/TradeTherapyBot
ExecStart=/usr/bin/python3 /path/to/TradeTherapyBot/main.py
User=root
```

**Должно быть (если бот в `/opt/tradetherapybot`):**
```
WorkingDirectory=/opt/tradetherapybot
ExecStart=/usr/bin/python3 /opt/tradetherapybot/main.py
User=root
```

**Важно:** 
- Замените `/opt/tradetherapybot` на реальный путь, где находится ваш бот
- Для `ExecStart` проверьте путь к Python: `which python3` (выполните эту команду, чтобы узнать путь)

### 5.4. Сохраните файл

В nano: `Ctrl+O`, `Enter`, `Ctrl+X`

### 5.5. Проверьте, что файл скопирован

```bash
ls -la /etc/systemd/system/tradetherapybot.service
```

## Шаг 6: Активация и запуск

### 6.1. Перезагрузите конфигурацию systemd

```bash
sudo systemctl daemon-reload
```

### 6.2. Включите автозапуск бота

```bash
sudo systemctl enable tradetherapybot.service
```

Эта команда скажет системе автоматически запускать бота при загрузке сервера.

### 6.3. Запустите бота сейчас

```bash
sudo systemctl start tradetherapybot.service
```

### 6.4. Проверьте статус

```bash
sudo systemctl status tradetherapybot.service
```

Если всё хорошо, вы увидите что-то вроде:
```
● tradetherapybot.service - Trade Therapy Bot
   Loaded: loaded (/etc/systemd/system/tradetherapybot.service; enabled)
   Active: active (running) since ...
```

## Шаг 7: Просмотр логов

### В реальном времени:
```bash
sudo journalctl -u tradetherapybot.service -f
```

Для выхода из просмотра логов нажмите `Ctrl+C`

### Последние 50 строк логов:
```bash
sudo journalctl -u tradetherapybot.service -n 50
```

## Управление ботом

### Остановить бота:
```bash
sudo systemctl stop tradetherapybot.service
```

### Запустить бота:
```bash
sudo systemctl start tradetherapybot.service
```

### Перезапустить бота:
```bash
sudo systemctl restart tradetherapybot.service
```

### Проверить статус:
```bash
sudo systemctl status tradetherapybot.service
```

### Отключить автозапуск (если нужно):
```bash
sudo systemctl disable tradetherapybot.service
```

## Проверка работы

1. Проверьте статус: `sudo systemctl status tradetherapybot.service`
2. Посмотрите логи: `sudo journalctl -u tradetherapybot.service -n 50`
3. Попробуйте написать боту в Telegram `/start`

## Если что-то не работает

1. **Бот не запускается:**
   - Проверьте логи: `sudo journalctl -u tradetherapybot.service -n 100`
   - Проверьте, что файл `env_vars.txt` настроен правильно
   - Проверьте пути в service файле

2. **Ошибка "Permission denied":**
   - Проверьте права доступа: `ls -la /opt/tradetherapybot`
   - Возможно нужно: `sudo chown -R $USER:$USER /opt/tradetherapybot`

3. **Python не найден:**
   - Проверьте путь: `which python3`
   - Обновите `ExecStart` в service файле правильным путем

4. **Бот падает:**
   - Смотрите логи: `sudo journalctl -u tradetherapybot.service -f`
   - Проверьте, что все зависимости установлены: `pip3 list`

## Готово!

Теперь бот работает автоматически и будет запускаться при перезагрузке сервера.
