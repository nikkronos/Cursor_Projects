# Диагностика ошибки запуска бота

## Проблема
Бот не запускается, сервис в состоянии "activating (auto-restart)" с ошибкой `status=1/FAILURE`.

## Шаг 1: Проверка логов

Выполните на сервере:

```bash
journalctl -u tradetherapybot.service -n 100 --no-pager
```

Или более детально:

```bash
journalctl -u tradetherapybot.service -n 200 --no-pager | tail -50
```

**Что искать:**
- Ошибки с `ERROR` или `CRITICAL`
- Сообщения о проблемах с токеном
- Ошибки импорта модулей
- Ошибки работы с БД

## Шаг 2: Проверка файла env_vars.txt

```bash
cd /opt/tradetherapybot
cat env_vars.txt
```

**Проверьте:**
- ✅ Нет лишних пробелов после `=`
- ✅ Нет пустых строк между переменными
- ✅ Токен записан полностью (без переносов строк)
- ✅ Формат правильный: `BOT_TOKEN=8585081841:AAFbEB8LAKB5yT_YPpEg0WxbPcZFxz5ZiLs`

**Правильный формат:**
```
BOT_TOKEN=8585081841:AAFbEB8LAKB5yT_YPpEg0WxbPcZFxz5ZiLs
ADMIN_ID=135366416
GROUP_CHAT_ID=-1003180666952
```

**Неправильно (с пробелами):**
```
BOT_TOKEN = 8585081841:AAFbEB8LAKB5yT_YPpEg0WxbPcZFxz5ZiLs  ❌
```

## Шаг 3: Проверка прав доступа

```bash
cd /opt/tradetherapybot
ls -la env_vars.txt
ls -la bot.db
ls -la logs/
```

**Должно быть:**
- `env_vars.txt` - читаемый
- `bot.db` - читаемый/записываемый
- `logs/` - директория существует и доступна для записи

## Шаг 4: Проверка Python и зависимостей

```bash
cd /opt/tradetherapybot
python3 --version
python3 -c "import telebot; print('OK')"
python3 -c "import dotenv; print('OK')"
python3 -c "import apscheduler; print('OK')"
```

Если какая-то библиотека не найдена:

```bash
pip3 install -r requirements.txt --break-system-packages
```

## Шаг 5: Запуск вручную для диагностики

```bash
cd /opt/tradetherapybot
python3 main.py
```

Это покажет ошибку напрямую в терминале.

**Прервать:** `Ctrl+C`

## Шаг 6: Проверка файла логов

```bash
cd /opt/tradetherapybot
tail -50 logs/bot.log
```

## Частые проблемы и решения

### Проблема 1: "Bot token not found"
**Причина:** Токен не загружается из env_vars.txt

**Решение:**
1. Проверьте путь к файлу: `pwd` (должно быть `/opt/tradetherapybot`)
2. Проверьте содержимое: `cat env_vars.txt`
3. Убедитесь, что нет пробелов: `BOT_TOKEN=токен` (не `BOT_TOKEN = токен`)

### Проблема 2: "ModuleNotFoundError"
**Причина:** Не установлены зависимости

**Решение:**
```bash
pip3 install -r requirements.txt --break-system-packages
```

### Проблема 3: "Permission denied"
**Причина:** Нет прав на запись

**Решение:**
```bash
chmod 644 env_vars.txt
chmod 666 bot.db  # если файл существует
chmod 755 logs/
```

### Проблема 4: "Invalid token"
**Причина:** Неправильный токен

**Решение:**
1. Проверьте токен в @BotFather
2. Убедитесь, что скопирован полностью
3. Проверьте, нет ли лишних символов

### Проблема 5: Ошибка при инициализации БД
**Причина:** Проблемы с правами на запись

**Решение:**
```bash
cd /opt/tradetherapybot
chmod 666 bot.db  # если файл существует
# или удалите и создайте заново
rm bot.db
python3 -c "from database import init_db; init_db()"
```

## После исправления

```bash
systemctl restart tradetherapybot.service
systemctl status tradetherapybot.service
```

Если всё ок, должно быть: `active (running)`
