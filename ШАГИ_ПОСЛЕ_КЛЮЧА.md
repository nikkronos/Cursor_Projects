# Что делать после создания SSH ключа

## ✅ ШАГ 1: Добавить публичный ключ на сервер

### 1.1. Скопировать публичный ключ

Из PowerShell (где вы запускали скрипт) скопируйте **публичный ключ** - это одна строка, которая начинается с `ssh-ed25519`.

**Пример:**
```
ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIMLMrYP43Kf4Cbk2WDRJ1xkBktKxB/bx95Ydj1dI4x8o github-actions-deploy
```

### 1.2. Подключиться к серверу

1. Откройте **НОВОЕ** окно PowerShell (не закрывайте то, где скрипт!)
   - Нажмите **Win + R**
   - Введите `powershell`
   - Нажмите **Enter**

2. Выполните команду:
   ```powershell
   ssh root@81.200.146.32
   ```

3. Введите пароль от сервера (который вы используете для подключения)

### 1.3. На сервере выполнить команды

После подключения к серверу, выполните команды **по очереди** (каждую отдельно, нажимая Enter после каждой):

**Команда 1:** Создать папку для ключей
```bash
mkdir -p ~/.ssh
```

**Команда 2:** Добавить публичный ключ (простой способ)

Сначала скопируйте ваш публичный ключ из PowerShell (где вы запускали скрипт).

Затем выполните эту команду на сервере, **заменив `ВАШ_ПУБЛИЧНЫЙ_КЛЮЧ`** на ваш реальный ключ:

```bash
echo 'ВАШ_ПУБЛИЧНЫЙ_КЛЮЧ' >> ~/.ssh/authorized_keys
```

**Пример:** Если ваш ключ `ssh-ed25519 AAAAC3...`, то команда будет:
```bash
echo 'ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIMLMrYP43Kf4Cbk2WDRJ1xkBktKxB/bx95Ydj1dI4x8o github-actions-deploy' >> ~/.ssh/authorized_keys
```

**Важно:** Скопируйте весь ключ целиком (одну строку) и вставьте его вместо `ВАШ_ПУБЛИЧНЫЙ_КЛЮЧ`.

---

### Альтернативный способ (если первый не работает):

**Команда 2 (альтернатива):** Использовать редактор
```bash
nano ~/.ssh/authorized_keys
```

**Что делать в редакторе:**
1. Прокрутите мышкой до самого конца файла (если там что-то есть)
2. Или просто нажмите **Enter** несколько раз, чтобы перейти в конец
3. **Вставьте** публичный ключ (Ctrl + V или правая кнопка мыши → Вставить)
4. Нажмите **Enter** еще раз (новая строка после ключа)
5. Сохранить: нажмите **Ctrl + O**, затем **Enter**
6. Выйти: нажмите **Ctrl + X**

### 1.5. Установить права доступа

Выполните команду:
```bash
chmod 600 ~/.ssh/authorized_keys
```

### 1.6. Выйти с сервера

Выполните:
```bash
exit
```

**Готово!** Публичный ключ добавлен на сервер.

## ✅ ШАГ 2: Добавить приватный ключ в GitHub Secrets

### 2.1. Открыть GitHub

1. Откройте браузер
2. Перейдите на https://github.com
3. Войдите в свой аккаунт
4. Откройте ваш репозиторий (TradeTherapyBot)

### 2.2. Перейти в настройки секретов

1. В репозитории нажмите на вкладку **Settings** (вверху справа, рядом с Code, Issues, Pull requests)
2. В левом меню найдите **Secrets and variables**
3. Нажмите **Actions**
4. Нажмите кнопку **New repository secret** (зеленая кнопка справа)

### 2.3. Добавить приватный ключ

1. В поле **Name** введите: `SSH_PRIVATE_KEY`

2. В поле **Secret** вставьте **ВЕСЬ приватный ключ** из PowerShell:
   - Начните с `-----BEGIN OPENSSH PRIVATE KEY-----`
   - Завершите `-----END OPENSSH PRIVATE KEY-----`
   - Скопируйте **ВСЕ строки** между ними (включая эти две строки)

3. Нажмите кнопку **Add secret** (зеленая кнопка внизу)

**Готово!** Первый секрет добавлен.

## ✅ ШАГ 4: Проверить работу (тестовый деплой)

После добавления всех секретов нужно протестировать, что всё работает.

### 4.1. Вернуться в PowerShell в папке проекта

Если PowerShell был закрыт, откройте его заново и выполните:
```powershell
cd "C:\Users\krono\OneDrive\Рабочий стол\Cursor_test\TradeTherapyBot"
```

### 4.2. Сделать тестовый коммит

Выполните команды **по очереди** (каждую отдельно):

```powershell
git add .
```

```powershell
git commit -m "ci: настройка автоматического деплоя"
```

```powershell
git push origin main
```

**Что должно произойти:**
- Код отправится в GitHub
- GitHub автоматически запустит тесты
- Если тесты пройдут, автоматически запустится деплой

### 4.3. Проверить результат на GitHub

1. Откройте ваш репозиторий на GitHub
2. Нажмите на вкладку **Actions** (вверху)
3. Вы увидите два процесса:
   - **Run Tests** - должен быть зеленым ✅
   - **Deploy to Server** - должен быть зеленым ✅

Если оба зеленые - **всё работает!** 🎉

## ❓ Если что-то пошло не так

### Проблема: "Permission denied" при подключении к серверу
**Решение:** 
- Проверьте, что публичный ключ правильно добавлен в `~/.ssh/authorized_keys`
- Проверьте права доступа: `chmod 600 ~/.ssh/authorized_keys`

### Проблема: "Деплой не запустился"
**Решение:**
1. Проверьте, что все 6 секретов добавлены в GitHub
2. Проверьте логи в разделе Actions на GitHub (там будет написано, какая ошибка)

### Проблема: "SSH connection failed"
**Решение:**
1. Проверьте, что публичный ключ добавлен на сервер
2. Попробуйте подключиться вручную: `ssh root@81.200.146.32`
