## 🚀 Как это работает теперь

### При каждом изменении кода:

1. **Вы редактируете код** локально
2. **Вы делаете коммит и push:**
   ```bash
   git add .
   git commit -m "описание изменений"
   git push origin main
   ```
3. **Автоматически:**
   - ✅ Запускаются тесты
   - ✅ Если тесты прошли → код обновляется на сервере
   - ✅ Создается бэкап БД
   - ✅ Обновляются зависимости
   - ✅ Бот перезапускается

**Вам больше не нужно вручную загружать файлы на сервер!**

## 📚 Полезные команды

### На локальном компьютере:

```bash

git commit -m "описание изменений"
git push origin main
```

### На сервере (если нужно проверить бота):

```bash
# Подключиться к серверу
ssh root@81.200.146.32

# Проверить статус бота
systemctl status tradetherapybot.service

# Посмотреть логи
journalctl -u tradetherapybot.service -n 50 --no-pager

# Перезапустить бота (обычно не нужно - происходит автоматически)
sudo systemctl restart tradetherapybot.service
```

## 📞 Если что-то пойдёт не так

### Проблема: Бот не работает

1. Проверьте статус на сервере: `systemctl status tradetherapybot.service`
2. Посмотрите логи: `journalctl -u tradetherapybot.service -n 50`
3. Проверьте файл `env_vars.txt`: `cat /opt/tradetherapybot/env_vars.txt`

### Проблема: Деплой не запускается

1. Проверьте GitHub Actions: есть ли ошибки?
2. Проверьте, что вы делаете push в ветку `main`
3. Проверьте, что тесты проходят
