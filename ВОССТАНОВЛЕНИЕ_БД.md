# Восстановление базы данных после git reset

## Проблема

При выполнении `git reset --hard` база данных была удалена, и бэкап тоже исчез.

## Решения

### Решение 1: Использовать бэкап из папки backups (если есть)

На сервере проверьте:

```bash
cd /opt/tradetherapybot
ls -la backups/
```

Если там есть файлы бэкапов, восстановите последний:

```bash
cp backups/bot_backup_*.db bot.db
```

Или найдите самый свежий:

```bash
ls -lt backups/ | head -2
cp backups/САМЫЙ_СВЕЖИЙ_ФАЙЛ.db bot.db
```

### Решение 2: Получить бэкап через команду бота

Если бот еще работает, используйте команду `/backup` в Telegram, чтобы получить актуальную базу данных.

### Решение 3: Если база данных потеряна

Если бэкапа нет, придется создать новую базу данных (потеря данных!). Бот создаст новую при первом запуске.

---

## Также нужно переключиться на ветку main

На сервере выполните:

```bash
cd /opt/tradetherapybot
git checkout -b main origin/main
```

Или:

```bash
git branch -M main
git checkout main
```
