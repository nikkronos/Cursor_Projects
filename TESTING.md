# Инструкция по тестированию

## Автоматическое тестирование (через AI агента)

Я проведу тестирование через специальный скрипт `test_manual.py`, который проверяет:
- Импорты всех модулей
- Функции валидации
- Утилиты (retry механизм)
- Структуру базы данных

## Ручное тестирование

### Вариант 1: Простой скрипт (без pytest)

Запустите:
```bash
python test_manual.py
```

Этот скрипт проверяет базовую функциональность без необходимости устанавливать pytest.

### Вариант 2: С pytest (если установлен)

Если у вас установлен pytest (есть в requirements.txt), можно запустить полные тесты:

```bash
# Установить pytest (если еще не установлен)
pip install pytest

# Запустить все тесты
pytest tests/ -v

# Запустить конкретный файл тестов
pytest tests/test_validators.py -v
pytest tests/test_utils_retry.py -v
pytest tests/test_database_indexes.py -v
```

## Что тестируется

### 1. Валидаторы (`tests/test_validators.py`)
- Валидация ID пользователя
- Валидация количества дней
- Валидация текстовых полей
- Проверка граничных случаев

### 2. Утилиты (`tests/test_utils_retry.py`)
- Механизм retry при временных ошибках
- Экспоненциальная задержка
- Обработка rate limiting (429 ошибки)
- Функция safe_send_message

### 3. База данных (`tests/test_database_indexes.py`)
- Создание индексов при инициализации
- Миграция существующих БД
- Проверка использования индексов в запросах

## Проверка линтером

Линтер проверяет код на ошибки автоматически. Если нужно проверить вручную:

В VS Code / Cursor обычно линтер работает автоматически. Или можно использовать:
```bash
python -m py_compile validators.py utils.py database.py
```

## Рекомендации

1. **Перед коммитом:** Запустите `python test_manual.py` или `pytest tests/`
2. **После изменений:** Проверьте, что все тесты проходят
3. **При проблемах:** Проверьте логи в `logs/bot.log`
